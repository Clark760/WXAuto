# 编辑器 EXE 打包方式

本文档说明如何把 `tools/mod_data_editor` 打包成可点击运行的 Windows `exe`。

## 1. 前置条件

1. 已安装 Python（建议 3.10+）
2. 在项目根目录可执行 `python`
3. 可联网安装依赖（用于安装 `pyinstaller`）
4. 脚本会自动安装 `pystray` 与 `pillow`（用于托盘功能）

## 2. 一键打包（推荐）

### PowerShell

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\mod_data_editor\build_exe.ps1
```

### CMD

```bat
tools\mod_data_editor\build_exe.bat
```

打包完成后输出目录：

```text
tools\mod_data_editor\dist\WXAutoModDataEditor\
```

可执行文件：

```text
tools\mod_data_editor\dist\WXAutoModDataEditor\WXAutoModDataEditor.exe
```

## 3. 当前脚本已包含的保护逻辑

- 自动删除旧的 `tools/mod_data_editor/WXAutoModDataEditor.spec`
- `--add-data` 使用绝对路径，避免相对路径重复拼接
- 任一步失败都会返回错误退出码，不再出现“报错但显示 Done”
- 打包结束后会检查 exe 是否真实存在
- EXE 启动时会显示控制窗口，可最小化到托盘并可从托盘重新打开

## 4. 手动打包命令（等价）

```powershell
python -m pip install -U pyinstaller
python -m PyInstaller `
  --noconfirm `
  --clean `
  --name WXAutoModDataEditor `
  --onedir `
  --noconsole `
  --distpath "tools/mod_data_editor/dist" `
  --workpath "tools/mod_data_editor/build" `
  --specpath "tools/mod_data_editor" `
  --add-data "<绝对路径>\\index.html;mod_data_editor_static" `
  --add-data "<绝对路径>\\styles.css;mod_data_editor_static" `
  --add-data "<绝对路径>\\app.js;mod_data_editor_static" `
  tools/mod_data_editor/server.py
```

## 5. 常见问题

1. 出现 `Unable to find ... tools/mod_data_editor/tools/mod_data_editor/index.html`  
原因：旧 `.spec` 里是相对路径，且被重复复用。  
解决：删除 `tools/mod_data_editor/WXAutoModDataEditor.spec` 后重新打包，或直接使用新的打包脚本。

2. 提示找不到项目根目录  
请在项目目录下运行，或启动 exe 时追加：

```powershell
.\WXAutoModDataEditor.exe --project-root "C:\Your\WXAuto\Path"
```

3. 修改前端后 exe 无变化  
删除 `tools/mod_data_editor/build` 与 `tools/mod_data_editor/dist` 后重打包。
