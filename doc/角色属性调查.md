# 角色属性调查（M5）

## 1. 本次已落地修改

1. `MP` 不再随升星变化。  
   实现：`scripts/data/unit_data.gd` 的 `build_runtime_stats` 已将 `mp` 从升星放大列表移除。
2. 暴击率上限改为 `100%`。  
   实现：`scripts/unit/unit_combat.gd::_calc_crit_rate()` 上限由 `0.8` 改为 `1.0`。
3. 增加攻击速度上限。  
   实现：`scripts/unit/unit_combat.gd` 新增 `max_attack_speed_per_sec`（默认 `4.0`），最终攻击间隔下限改为 `1 / max_attack_speed_per_sec`。

## 2. 每个属性在战斗中参与了什么判断

| 属性 | 参与模块 | 作用 |
|---|---|---|
| HP | `UnitCombat`、`GongfaManager` 触发判定 | 生存与死亡判定；`hp_below`/`on_hp_below` 等触发条件；治疗、复活、护盾结算都围绕 HP。 |
| MP | `UnitCombat`、`GongfaManager` | 自动技能释放资源（`current_mp >= skill_mp_cost`）；受击/普攻/回蓝效果改变 MP；触发条目的 `mp_cost` 也用当前 MP 判定。 |
| ATK | `UnitCombat` | 外功攻击面板；普攻优先外功轨道时作为主攻值；参与外功伤害公式。 |
| IAT | `UnitCombat` | 内功攻击面板；技能优先内功轨道时作为主攻值；参与内功伤害公式。 |
| DEF | `UnitCombat` | 作为外功防御参与减伤（外功轨道命中时读取 `def`）。 |
| IDR | `UnitCombat` | 作为内功防御参与减伤（内功轨道命中时读取 `idr`）。 |
| SPD | `UnitCombat` | 影响闪避率、攻击间隔（攻速）；不影响位移速度（当前位移由 `UnitMovement.base_move_speed` 固定控制）。 |
| RNG | `UnitCombat`、`CombatTargeting`、`GongfaManager` | 普攻射程格数；默认技能施法距离（skill 未配置 `range` 时回退到 `rng`）。 |
| WIS | `UnitCombat` | 暴击率、暴击伤害倍率。 |

补充：
- `MOV` 已在当前代码链路中移除，不参与战斗判定。
- 升星当前只放大：`hp/atk/iat/def/idr`。

## 3. SPD 与 WIS 的阈值影响

## 3.1 SPD（速度）

当前攻击间隔公式：

```text
base_interval = clamp(1.8 - spd / 120, 0.24, 2.2)
speed_bonus  = clamp(attack_speed_bonus, -0.8, 0.9)
attack_interval = clamp(base_interval * (1 - speed_bonus), 1/max_attack_speed_per_sec, 3.0)
```

默认 `max_attack_speed_per_sec = 4.0`，即攻击间隔最低 `0.25s`（4次/秒）。

无额外攻速加成时（`attack_speed_bonus=0`）：

| SPD | 攻击间隔(s) | 攻击频率(次/秒) |
|---:|---:|---:|
| 80 | 1.133 | 0.88 |
| 100 | 0.967 | 1.03 |
| 120 | 0.800 | 1.25 |
| 140 | 0.633 | 1.58 |
| 160 | 0.467 | 2.14 |
| 180 | 0.300 | 3.33 |
| 200 | 0.250 | 4.00（触顶） |

结论：
- 纯白板下，`SPD≈186+` 会触及攻速上限（再加 SPD 收益趋近 0）。
- 现有数据中最高基础 `SPD=149`，不会在白板状态触顶。

闪避率（防守方）公式：

```text
dodge = clamp((target_spd - attacker_spd) / (target_spd + 100), 0, 0.4) + dodge_bonus
最终再 clamp 到 [0, 0.75]
```

结论：
- 仅当防守方 `SPD > 攻击方 SPD` 时，基础闪避开始大于 0。
- 基础公式上限 40%，再叠加 `dodge_bonus` 才可继续提高到 75% 总上限。

## 3.2 WIS（悟性）

暴击率：

```text
crit_rate = clamp(0.05 + wis * 0.001 + crit_bonus, 0.05, 1.0)
```

暴击伤害倍率：

```text
crit_mult = 1.5 + wis * 0.0005 + crit_damage_bonus
```

无额外暴击加成时：

| WIS | 暴击率 | 暴伤倍率 |
|---:|---:|---:|
| 60 | 11% | 1.53 |
| 100 | 15% | 1.55 |
| 150 | 20% | 1.575 |
| 200 | 25% | 1.60 |
| 300 | 35% | 1.65 |
| 500 | 55% | 1.75 |
| 950 | 100%（触顶） | 1.975 |

结论：
- WIS 对暴击率是线性增益（每 +1 WIS = +0.1% 暴击率）。
- WIS 对暴伤是较慢线性增益（每 +1 WIS = +0.05% 暴伤倍率）。

## 4. 各品质推荐基础属性区间（基于当前战斗系统）

以下是面向数据策划的“推荐基础值区间”（单位基础面板 `base_stats`），用于保证成长梯度和攻速收益稳定：

| quality | HP | MP | ATK | IAT | DEF | IDR | SPD | RNG | WIS |
|---|---:|---:|---:|---:|---:|---:|---:|---:|---:|
| white | 680-950 | 60-110 | 45-90 | 45-95 | 20-65 | 20-55 | 65-95 | 1-3 | 40-80 |
| green | 840-1150 | 70-130 | 60-110 | 60-110 | 28-70 | 25-65 | 75-105 | 1-3 | 50-90 |
| blue | 980-1320 | 85-145 | 75-130 | 75-130 | 35-85 | 30-75 | 85-118 | 1-4 | 60-105 |
| purple | 1180-1680 | 100-190 | 90-170 | 90-160 | 48-105 | 40-90 | 95-130 | 1-4 | 80-150 |
| orange | 1550-2700 | 145-300 | 115-260 | 130-240 | 70-165 | 60-145 | 105-150 | 1-4 | 110-230 |

攻速相关重点建议：

1. 基础 `SPD` 不建议常态超过 `150`。  
2. 将“触顶攻速”主要留给短时 Buff（而不是常驻白板），避免普攻动画与结算过密导致观感噪声。  
3. 若某职业定位是“极致连击”，建议通过 `attack_speed_bonus` 峰值窗口实现，而非长期堆高白板 SPD。  

## 5. 当前系统下，攻速上限设置多少更合理

当前默认上限改为 `4.0 次/秒`（`0.25s` 间隔）后：

1. 视觉可读性更稳定，不会出现过快抖动与日志刷屏。
2. 触发器风暴风险显著小于旧的超高速上限（旧值等效最高可达 `12.5 次/秒`）。
3. 结合现有数据（基础 SPD 最高 149、单条 `attack_speed_bonus` 当前最大 0.16），多数单位不会轻易常驻触顶。

对应属性值参考：

1. 无攻速加成时，约 `SPD >= 186` 才触顶。  
2. 带 `+30%` 攻速加成时，约 `SPD >= 177` 触顶。  
3. 以当前数据常见加成（约 `+6% ~ +16%`）来看，触顶阈值仍在高 SPD 区间。  

## 6. 是否应修改攻速公式

结论：**本轮建议“只改上限，不改主公式形态”**，并已执行。理由：

1. 主公式仍线性、直观，策划可控性高。  
2. 改为 4.0 次/秒上限后，已经解决“极端高攻速导致战斗节奏失真”的主要问题。  
3. 若后续出现“高 SPD 区间收益仍过陡”再考虑二次改造（例如分段函数），当前无需立即引入复杂公式。  

## 7. Effect 攻防属性上限建议（参考同类PVE自走棋）

本节用于补充 `effect` 在攻防属性上的策划预算，口径采用五项拆分：`HP / ATK / IAT / DEF / IDR`。  
这些上限是结合升星缩放（1★/2★/3★）后的平衡建议，用于防止中后期爆表。  
说明：这是策划推荐上限，不是运行时硬限制，当前代码不会自动 clamp 到本表数值。

### 7.1 参考基线（简述）

参考范式：`TFT`、`酒馆战棋`、`刀塔自走棋`。  
提炼原则：
1. 核心攻防可以成长，但常驻乘区必须受控。
2. 防御属性存在边际递减，仍需预算上限，避免战斗时长失控。
3. 爆发窗口允许更高上限，但必须与常驻上限分层管理。

### 7.2 上限总表（可直接作为策划约束）

`A. 常驻生效（被动/常驻光环/长时Buff）`

| 属性 | 单条 `stat_add` 上限 | 单条 `stat_percent` 上限 | 单位累计 `stat_add` 上限 | 单位累计 `stat_percent` 上限 |
|---|---:|---:|---:|---:|
| HP | +800 | +20% | +1800 | +35% |
| ATK | +70 | +18% | +160 | +30% |
| IAT | +70 | +18% | +160 | +30% |
| DEF | +50 | +14% | +120 | +25% |
| IDR | +50 | +14% | +120 | +25% |

`B. 短时爆发（持续时间较短、窗口型Buff）`

| 属性 | 单条 `stat_add` 上限 | 单条 `stat_percent` 上限 | 单位累计 `stat_add` 上限 | 单位累计 `stat_percent` 上限 |
|---|---:|---:|---:|---:|
| HP | +1200 | +30% | +2800 | +55% |
| ATK | +110 | +24% | +240 | +45% |
| IAT | +110 | +24% | +240 | +45% |
| DEF | +80 | +20% | +180 | +38% |
| IDR | +80 | +20% | +180 | +38% |

现有数据对齐说明：当前仓库存在少量 `ATK stat_percent = 17.1%/18%` 的常驻配置，故常驻单条上限采用 `18%` 以避免与既有数据冲突。

### 7.3 组合约束（防叠乘穿透预算）

统一校验式（文档规则）：

```text
effective_bonus_ratio = ((base_star_scaled + add_total) * (1 + percent_total) / base_star_scaled) - 1
```

推荐组合上限：
1. 常驻：`HP ≤ +60%`，`ATK/IAT ≤ +55%`，`DEF/IDR ≤ +45%`
2. 短时：`HP ≤ +95%`，`ATK/IAT ≤ +85%`，`DEF/IDR ≤ +70%`

### 7.4 与升星缩放的关系

说明：升星放大的是基础攻防（`hp/atk/iat/def/idr`），`stat_add` 为固定量，`stat_percent` 为比例量。  
结论：
1. 低星期：`stat_add` 体感更强。
2. 高星期：`stat_percent` 体感更稳定。
3. 设计建议：跨星级稳定收益优先放在 `stat_percent`，前期托底收益优先放在 `stat_add`。

## 8. MP / SPD / WIS 的 Effect 预算建议

本节基于当前机制：
1. `MP` 不随升星变化。
2. `SPD` 受攻速上限约束（当前默认 `4.0 次/秒`）。
3. `WIS` 影响暴击率/暴伤，暴击率总上限 `100%`。

口径统一为：
1. `常驻` / `短时爆发` 两档。
2. 每档包含 `单条 effect 上限` 与 `单单位累计上限`。
3. 这是策划预算建议，不是运行时代码硬限制。

| 维度 | effect | 常驻（单条 \| 累计） | 短时爆发（单条 \| 累计） |
|---|---|---|---|
| MP | `stat_add(mp)` | `+80 \| +180` | `+140 \| +320` |
| MP | `stat_percent(mp)` | `+15% \| +30%` | `+25% \| +50%` |
| MP | `mp_regen_add`（每秒） | `+4/s \| +8/s` | `+8/s \| +16/s` |
| MP | `mp_on_kill` | `+15 \| +30` | `+25 \| +50` |
| SPD | `stat_add(spd)` | `+25 \| +45` | `+40 \| +80` |
| SPD | `stat_percent(spd)` | `+12% \| +22%` | `+20% \| +40%` |
| SPD | `attack_speed_bonus` | `+0.12 \| +0.22` | `+0.22 \| +0.40` |
| SPD | `dodge_bonus` | `+0.06 \| +0.12` | `+0.12 \| +0.22` |
| WIS | `stat_add(wis)` | `+35 \| +70` | `+60 \| +120` |
| WIS | `stat_percent(wis)` | `+12% \| +25%` | `+22% \| +45%` |
| WIS | `crit_bonus` | `+0.10 \| +0.20` | `+0.20 \| +0.35` |
| WIS | `crit_damage_bonus` | `+0.20 \| +0.35` | `+0.35 \| +0.60` |

补充建议：
1. `attack_speed_bonus` 常驻尽量不超过 `0.22`，避免普遍贴近攻速上限后 SPD 收益失真。
2. `crit_bonus + wis * 0.001` 建议重点监控，常驻配置尽量让大多数单位处于 `15%~45%` 暴击区间。
3. `mp_regen_add` 是最容易导致技能循环过快的参数，常驻推荐优先控制在 `+4/s` 附近。
