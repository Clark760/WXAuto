# 角色属性调查

## 1. 每个属性在战斗中参与了什么判断

| 属性 | 参与模块 | 作用 |
|---|---|---|
| HP | `UnitCombat`、`GongfaManager` 触发判定 | 生存与死亡判定；`hp_below`/`on_hp_below` 等触发条件；治疗、复活、护盾结算都围绕 HP。 |
| MP | `UnitCombat`、`GongfaManager` | 受击/普攻/回蓝效果改变 MP；触发条目的 `mp_cost` 也用当前 MP 判定。普通攻击链不再内置自动技能释放。 |
| ATK | `UnitCombat` | 外功攻击面板；**普攻时当 `atk >= iat * 0.8` 走外功轨道**。参与伤害公式中的 `offense` 项。 |
| IAT | `UnitCombat` | 内功攻击面板；**普攻时当 `atk < iat * 0.8` 回退走内功轨道**。参与伤害公式中的 `offense` 项。 |
| DEF | `UnitCombat` | 作为外功防御参与减伤（外功轨道命中时读取 `def`）。伤害公式：`raw = offense * multiplier * (100 / (100 + def)) * random`。 |
| IDR | `UnitCombat` | 作为内功防御参与减伤（内功轨道命中时读取 `idr`）。公式同 DEF，替换为 `idr`。 |
| SPD | `UnitCombat` | 影响闪避率（作为防守方 SPD）、攻击间隔（攻速）；不影响位移速度（当前位移由 `UnitMovement.base_move_speed` 固定控制）。 |
| RNG | `UnitCombat`、`CombatTargeting`、`UnitAugmentSkillTargetService` | 普攻射程格数（`get_attack_range_cells()`）；默认技能施法距离（skill 未配置 `range` 时回退到 `rng`，见 `resolve_skill_cast_range_cells()`）。 |
| WIS | `UnitCombat` | 暴击率（线性 +0.2%/WIS）、暴击伤害倍率（线性 +0.05%/WIS）。 |

## 1.1额外属性表：
- `mp_regen_add`：+MP 每秒（由 `mp_regen_add` op 提供）；影响技能条目与其他 MP 机制的释放频率。
- `hp_regen_add`：+HP 每秒（`hp_regen_add` op）；影响生存与回血恢复速率。
- `damage_reduce_flat` / `damage_reduce_percent`：固定/百分比减伤（`damage_reduce_flat`、`damage_reduce_percent`）；直接降低承受伤害。
- `damage_amp_percent`：通用增伤（`damage_amp_percent`）；提升输出伤害。
- `damage_amp_vs_debuffed`：对被减益目标的增伤。
  > 实际代码拆分为两个独立字段——`damage_amp_vs_any_debuff`（目标有任意 debuff 时的增伤倍率）和 `damage_amp_vs_debuff_map`（按具体 debuff_id 分别指定的增伤 map）。`passive_effect_applier.gd` 中 op 名为 `damage_amp_vs_debuffed`，当 `require_debuff` 为空时写入 `damage_amp_vs_any_debuff`，有值时写入 map。
- `dodge_bonus`：闪避加成（`dodge_bonus`）；在闪避公式的基础值之上叠加，最终 clamp 到 [0, 0.75]。
- `crit_bonus` / `crit_damage_bonus`：暴击率与暴伤（`crit_bonus`、`crit_damage_bonus`）；影响暴击触发与伤害倍率。
- `vampire`：吸血比例（`vampire`）。
  > `vampire` **仅在普攻命中后生效，技能命中不触发吸血**。当前是由普攻链路在命中后调用吸血结算，clamp 范围 [0, 5.0]。
- `tenacity`：韧性（`tenacity`）；缩短 Debuff 时长，钳制到 [0.0, 0.9] 并最小保留 0.1s。
- `thorns_percent` / `thorns_flat`：反伤百分比/固定反伤（`thorns_percent`、`thorns_flat`）；攻击者受到反弹伤害。
- `shield_on_combat_start`：开场护盾（`shield_on_combat_start`）；战斗开始时授予护盾值。
- `execute_threshold`：斩杀阈值（`execute_threshold`）；低血目标触发斩杀逻辑。
- `healing_amp`：治疗加成（`healing_amp`）；由 `restore_hp()` 统一消费。
- `mp_on_kill`：击杀回蓝（`mp_on_kill`）；击杀时恢复 MP。
- `conditional_stat`：条件属性增益（`conditional_stat`）；按条件（如 `hp_below`）提供 stats。
- `attack_speed_bonus`：攻速加成（`attack_speed_bonus`）；影响攻击间隔与攻速上限计算。
- `range_add`：射程加成（`range_add`）；影响普攻/技能覆盖格数。

## 2. 完整战斗公式链

> 以下所有公式均从 `unit_combat.gd` 逐行提取，附代码行号。

### 2.1 伤害计算全流程

```text
┌─ 攻击方 ──────────────────────────────────────────────────────────┐
│ ① 轨道选择                                                       │
│   普攻: atk >= iat*0.8 → external(atk)  否则 internal(iat)       │
│                                                                   │
│ ② 基础伤害                                                       │
│   multiplier = 1.0                                                │
│   defense    = external ? target.def : target.idr                 │
│   random     = lerp(0.9, 1.1, rand())                             │
│   raw_damage = offense * multiplier * (100 / (100+defense)) * random │
│                                                                   │
│ ③ 暴击                                                           │
│   crit_rate = clamp(0.05 + wis*0.002 + crit_bonus, 0.05, 1.0)    │
│   crit_mult = 1.5 + wis*0.0005 + crit_damage_bonus               │
│   if crit → raw_damage *= crit_mult                               │
│                                                                   │
│ ④ 增伤                                                           │
│   raw_damage *= max(1 + damage_amp_percent, 0)                    │
│   raw_damage *= debuff_amp_ratio  (见下方拆解)                     │
│                                                                   │
│ ⑤ 斩杀                                                           │
│   threshold = clamp(execute_threshold, 0, 0.95)                   │
│   if target_hp/target_max_hp <= threshold → raw_damage *= 2.0     │
└───────────────────────────────────────────────────────────────────┘

┌─ 防守方 ──────────────────────────────────────────────────────────┐
│ ⑥ 闪避                                                           │
│   dodge = clamp((target_spd - attacker_spd)/(target_spd+100),0,0.4)│
│         + dodge_bonus                                              │
│   dodge = clamp(dodge, 0, 0.75)                                   │
│   if rand() < dodge → damage = 0, 跳过后续                        │
│                                                                   │
│ ⑦ 护盾吸收                                                       │
│   shield_absorbed = min(raw_damage, shield_hp)                    │
│   remaining = raw_damage - shield_absorbed                        │
│   if remaining <= 0 → 不扣 HP，不触发减伤/反伤                    │
│                                                                   │
│ ⑧ 减伤                                                           │
│   final = max(remaining - damage_reduce_flat, 0)                  │
│   reduce_ratio = clamp(damage_reduce_percent, -0.95, 0.95)        │
│   final *= (1 - reduce_ratio)                                     │
│                                                                   │
│ ⑨ 反伤（仅 external 轨道）                                       │
│   reflect = final * thorns_percent + thorns_flat                  │
│   → 对攻击方调用 receive_damage(reflect)                          │
│                                                                   │
│ ⑩ 受击回蓝                                                       │
│   target.mp += mp_gain_on_hit (基础 0 + modifier 叠加)            │
└───────────────────────────────────────────────────────────────────┘

┌─ 攻击后 ──────────────────────────────────────────────────────────┐
│ ⑪ 资源变化                                                       │
│   普攻命中: mp += mp_gain_on_attack (固定 15)                     │
│                                                                   │
│ ⑫ 吸血                                                           │
│   vampire_ratio = clamp(vampire, 0, 5.0)                          │
│   heal = dealt_damage * vampire_ratio → restore_hp                │
│                                                                   │
│ ⑬ 攻击冷却                                                       │
│   attack_cd = attack_interval                                     │
└───────────────────────────────────────────────────────────────────┘
```

### 2.2 debuff 增伤拆解

```text
debuff_amp_ratio = 1.0
// ① 任意 debuff 增伤
if damage_amp_vs_any_debuff > 0 且 target 有 debuff:
    ratio *= max(1 + damage_amp_vs_any_debuff, 0)
// ② 指定 debuff 增伤 (map)
for debuff_id in target.active_debuff_ids:
    if damage_amp_vs_debuff_map[debuff_id] exists:
        ratio *= max(1 + map[debuff_id], 0)
```

### 2.3 攻速公式

```text
base_interval = clamp(1.8 / (1 + spd/100), 0.24, 2.2)
speed_bonus   = clamp(attack_speed_bonus, -0.8, 0.9)
min_interval  = 1 / max_attack_speed_per_sec        (默认 = 0.25s)
attack_interval = clamp(base_interval * (1 - speed_bonus), min_interval, 3.0)
```

### 2.4 MP 循环公式

```text
初始 MP      = max_mp * 0.35
自然回蓝/s   = passive_mp_regen(2) + mp_regen_add
普攻回蓝     = 15 / 次
受击回蓝     = 0 / 次
击杀回蓝     = mp_on_kill

普通攻击不再内置技能释放判定
```

### 2.5 生命回复公式

```text
hp_regen/s = passive_hp_regen(0) + hp_regen_add
每逻辑帧: restore_hp(hp_regen * delta)   (仅正值有效)
回复事件每 0.5s 合并发射一次 (REGEN_HEAL_EMIT_INTERVAL)
```

## 3. 每属性详解：公式、范围、区间建议

### 3.1 HP（生命值）

| 项 | 值 |
|---|---|
| 公式影响 | `max_hp = runtime_stats.hp`；死亡判定 `current_hp <= 0` |
| 护盾独立 | `shield_hp` 独立于 HP，受击优先扣护盾 |
| 代码 clamp | `max_hp >= 1.0` |
| 现有数据范围 | white 697-942, green 1014-1280, blue 1158-1480, purple 1380-1780, orange 1620-2325 |

**建议基础区间**

| 品质 | 坦克 | 战士 | 射手 | 刺客 | 法师 | 牧师 |
|---|---:|---:|---:|---:|---:|---:|
| white | 820-950 | 760-900 | 680-800 | 700-830 | 690-820 | 720-860 |
| orange | 2300-2700 | 1950-2400 | 1550-1900 | 1650-2050 | 1650-2050 | 1750-2200 |

### 3.2 MP（内力）

| 项 | 值 |
|---|---|
| 公式影响 | 参与回蓝、受击回蓝与 trigger `mp_cost` 判定 |
| 初始比例 | 战斗开始时 `current_mp = max_mp * 0.35` |
| 代码 clamp | `max_mp >= 0.0` |
| 现有数据范围 | white 64-104, green 93-149, blue 118-169, purple 141-206, orange 154-286 |

> [!TIP]
> MP 仍然是 trigger `mp_cost` 与主动效果链的重要资源，但普通攻击不会再因为蓝量充足而自动改走“技能攻击”。

**建议基础区间**

| 品质 | 坦克 | 战士 | 射手 | 刺客 | 法师 | 牧师 |
|---|---:|---:|---:|---:|---:|---:|
| white | 70-95 | 65-90 | 60-85 | 60-85 | 85-110 | 90-110 |
| orange | 170-240 | 155-220 | 145-200 | 145-210 | 240-300 | 245-300 |

### 3.3 ATK / IAT（外功 / 内功攻击）

| 项 | 值 |
|---|---|
| 轨道选择 | 普攻: `atk >= iat*0.8` → 外功；否则内功 |
| 伤害公式 | `offense * multiplier * (100/(100+defense)) * random(0.9~1.1)` |
| 普攻倍率 | 1.0 |
| 现有数据 ATK | white 49-80, green 78-120, blue 96-149, purple 118-176, orange 116-224 |
| 现有数据 IAT | white 56-86, green 74-115, blue 93-141, purple 117-172, orange 139-233 |

> [!NOTE]
> 当 ATK 和 IAT 差距不大时（如 ATK=80, IAT=70），两条轨道都可能交替使用。设计上如果要让角色固定走某一轨道，**ATK 与 IAT 至少应保持 1.5 倍以上差距**。

### 3.4 DEF / IDR（外功 / 内功防御）

| 项 | 值 |
|---|---|
| 减伤公式 | `damage *= 100 / (100 + defense)`；DEF=100 → 外功减伤 50% |
| 边际递减 | DEF 从 0→100 减伤 0%→50%；100→200 只增 50%→67% |
| 额外减伤 | `damage_reduce_flat`（减固定量） + `damage_reduce_percent`（减百分比） |

**DEF/IDR 减伤参考表**

| DEF/IDR | 减伤率 | 备注 |
|---:|---:|---|
| 0 | 0% | |
| 50 | 33% | white 坦克上限区 |
| 100 | 50% | orange 战士/坦克区 |
| 150 | 60% | orange 坦克顶配 + buff |
| 200 | 67% | 超出正常范围 |

### 3.5 SPD（速度）

| 项 | 值 |
|---|---|
| 攻速公式 | `interval = clamp(1.8 / (1 + spd/100), 0.24, 2.2) * (1 - speed_bonus)` |
| 攻速上限 | `max_attack_speed_per_sec = 4.0`（间隔下限 0.25s） |
| 闪避公式 | `dodge = clamp((target_spd - atk_spd)/(target_spd+100), 0, 0.4) + dodge_bonus`, clamp [0, 0.75] |
| 不影响 | 移动速度（由 `UnitMovement.base_move_speed` 固定控制） |
| 现有数据 | white 72-96, green 83-111, blue 96-121, purple 108-134, orange 101-138 |

**SPD 攻速参考表**（无 `attack_speed_bonus`）

| SPD | 间隔(s) | 频率(/s) |
|---:|---:|---:|
| 80 | 1.00 | 1.00 |
| 100 | 0.90 | 1.11 |
| 120 | 0.82 | 1.22 |

> [!IMPORTANT]
> SPD 基础曲线在高区间改为平缓递减收益，不再像旧版那样在 186 附近很快触顶。攻速封顶主要由 `attack_speed_bonus` 与极高 SPD 共同触发。

### 3.6 RNG（射程）

| 项 | 值 |
|---|---|
| 普攻射程 | `max(int(rng + range_add), attack_range_min_cells)` 格 |
| 技能射程 | skill 配置 `range` > 0 时用配置值；否则退回 `rng` |
| 世界距离换算 | `range_cells * hex_size * 1.2`（容差系数） |
| 现有数据 | white 1-4, 所有品质 1-4 |

> [!TIP]
> RNG=1 为近战，RNG=3-4 为远程。远程单位天然多 2-3 帧安全输出窗口。RNG ≥ 5 在当前六角格规模下边际收益极低。

| 现有数据 | white 48-76, green 63-95, blue 79-118, purple 96-160, orange 113-224 |

## 5. 各职业推荐基础属性（base_stats）

> 以下是 `base_stats` 配表模板。现有数据（`bm5_units_batch*.json`）均落在这些区间内。

### 5.1 坦克（Tank）

设计要点：最高 HP/DEF/IDR，低 ATK/SPD/WIS。

| 品质 | HP | MP | ATK | IAT | DEF | IDR | SPD | RNG | WIS |
|---|---:|---:|---:|---:|---:|---:|---:|---:|---:|
| white | 820-950 | 70-95 | 45-62 | 45-60 | 45-65 | 38-55 | 65-80 | 1-2 | 40-55 |
| green | 980-1150 | 80-110 | 60-78 | 60-75 | 50-70 | 45-65 | 75-90 | 1-2 | 50-70 |
| blue | 1140-1320 | 95-125 | 75-92 | 75-95 | 62-85 | 55-75 | 85-100 | 1-2 | 60-82 |
| purple | 1460-1680 | 120-160 | 90-115 | 90-112 | 80-105 | 72-90 | 95-112 | 1-2 | 80-110 |
| orange | 2300-2700 | 170-240 | 115-150 | 145-175 | 130-165 | 115-145 | 105-125 | 1-2 | 110-160 |
