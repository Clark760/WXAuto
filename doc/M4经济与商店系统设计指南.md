# M4：经济与综合商店系统详细设计

## 1. 概述
在 M4 阶段，我们将建立包含玩家资产（银两/经验）和局内消费（综合商店）的核心系统。根据设计要求，**银两管理与门派等级经验系统将合并为一套基础经济数据系统**；同时，原本分散的**侠客商店、秘籍商店、装备商店将整合为一个统一的综合商店 UI 面板**。

## 2. 基础经济与经验数据系统

将银两和门派经验作为“玩家局内资产”统一管理，由数据管理器或专门的 `EconomyManager` 提供接口。

### 2.1 数据定义 (PlayerAsset)
```gdscript
var assets = {
    "silver": 0,           # 银两
    "level": 1,            # 门派等级
    "exp": 0,              # 当前等级经验
    "max_exp": 2,          # 升级所需经验
    "streak_count": 0,     # 连胜/连败场数 (正为胜, 负为败)
    "locked_shop": false   # 商店是否被锁定
}
```

### 2.2 核心接口
- `add_silver(amount: int)` / `spend_silver(amount: int) -> bool`
- `add_exp(amount: int)`: 经验达到界限时自动触发 `level_up()`
- `calculate_round_income() -> int`: 计算回合基础收入 + 利息(最高50的10%) + 连胜/败补偿
- 信号：`silver_changed`, `level_changed`, `exp_changed`

## 3. 综合商店系统架构

### 3.1 随机池机制
商店池子需要包含三种商品，由同一个 `ShopManager` 负责刷新：
1. **侠客池**：根据当前门派等级的概率表（例如 LV1: 100%白, 0%绿...）随机抽取5个侠客。
2. **秘籍池**：根据当前等级或回合数，从功法JSON中抽取3-5本功法。
3. **装备池**：根据当前等级或回合数，从装备JSON中抽取3-5件装备。

### 3.2 商店操作
- **刷新 (Refresh)**：消耗 2 银两，清空并重新随机生成三类商品，取消锁定状态。
- **锁定 (Lock)**：将当前商店所有商品锁定，下回合开始时**不自动刷新**。
- **购买 (Buy)**：检定银两 -> 扣除银两 -> 商品脱销 -> 发放侠客(至备战区)/秘籍(至仓库)/装备(至仓库)。

## 4. UI 样式规划（基于 M3 UI 设计）

综合商店 UI 应当融入 M3 的整体布局。因为 M3 已经由右侧常驻了“功法装备仓库”，底部常驻了“备战区”，因此商店最适合的位置是**取代底部备战区**或者作为**覆盖在战场区上方的全屏半透明面板**。

考虑到自走棋的习惯，**采用覆盖在战场区中下部的横向宽体面板**最为合适。

### 4.2 商品卡片设计
不同页签下的卡片 UI 略有不同，但尺寸和背景框统一：

- **侠客卡片**：显示头像、名字、势力/职业图标、星级（默认1星）、价格。悬停显示技能和羁绊 Tooltip。
- **秘籍卡片**：显示功法名称、品质色条、属性加成简述、价格。悬停显示原版 `ItemTooltip`。
- **装备卡片**：显示装备名称、品质色条、部位图标、价格。悬停显示原版 `ItemTooltip`。

### 4.3 UI 交互细节
1. **呼出方式**：底部的 ShopBar（原占位符）点击后弹出此面板。或者回合"布阵期"开始时自动弹出。
2. **关闭方式**：点击右上角[X]、按 ESC，或切换到交锋期时自动关闭。
3. **数据联动**：无论在哪个页签点击刷新，**所有页签的商品都会同时刷新**。锁定也将锁定所有页签。
4. **备战区联动**：买侠客时，如果底部备战席已满，应当提示“备战区已满，无法招募”。
5. **仓库联动**：买秘籍/装备时，直接进入右侧的 InventoryPanel 并触发其 UI 更新。

## 5. 修改清单指南给代码 AI
1. **新建/修改脚本**：创建并完善 `economy_manager.gd` 和 `shop_manager.gd`。
2. **开发商店 UI 节点**：在 `battlefield_m3.tscn` 中新建 `ShopLayer`，包含 `TabContainer` 实现上述布局。
3. **复用 Tooltip**：悬停商店里的秘籍和装备时，复用 `battlfield_m3.gd` 里的 `ItemTooltip`。
4. **连接槽位**：将 `BottomLayer` 下的银两/等级 Label 与新的 `EconomyManager` 的信号绑定。
