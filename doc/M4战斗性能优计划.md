# M4 战斗性能优化计划

## 1. 问题现象

1. **战斗开始瞬间 FPS 骤降**：按 F6 进入交锋期后前 1–3 秒帧率明显跌落。
2. **战斗持续越久越卡**：FPS 随战斗时间逐步下降。
3. **布阵期基线偏低**：空闲阶段 FPS 未稳定 60。

## 2. 战斗开始卡顿根因

追踪 `_start_combat()` 完整调用链后发现：**单帧内发生了 ~6 次 400+ 单位的全量遍历**，每次都含重型操作。

### 2.1 调用链详细拆解

```
_start_combat()
├─ _auto_deploy_from_bench(50)       # 遍历 bench + deploy
├─ _spawn_enemy_wave(200)            # 同步 instantiate 200 个场景 + texture load
│   └─ ×200: _deploy_enemy_unit_to_cell()
│       └─ _apply_unit_visual_presentation() + play_anim_state()
│
├─ gongfa_manager.prepare_battle()   # ★ 最大开销
│   ├─ ×400: apply_gongfa(unit, defer=true)
│   │   ├─ _build_unit_baseline_stats() → load("unit_data.gd") ← 每次都 load
│   │   ├─ ×N: _registry.get_gongfa() + effect.duplicate(true)
│   │   └─ ×N: skill_data.duplicate(true)
│   └─ _refresh_linkages(false)
│       ├─ _linkage_detector.detect_all()
│       │   └─ ×L×T: _match_linkage() → linkage.duplicate(true)
│       ├─ ×效果数: linkage_effect.duplicate(true)
│       ├─ ×400: _apply_state_to_unit()     ← 第二次全量 apply
│       │   ├─ baseline_stats.duplicate(true)
│       │   ├─ 4 层 apply_passive_effects()
│       │   ├─ combat.refresh_runtime_stats()
│       │   │   └─ combat.set_external_modifiers() → 再次 refresh
│       │   └─ movement.refresh_runtime_stats()
│       └─ linkage_changed.emit(get_active_linkages())
│           └─ ×L: item.duplicate(true)     ← 又一轮深拷贝
│
├─ combat_manager.start_battle()
│   ├─ _register_units() ×400
│   │   └─ _cache_components_for_unit() + combat.prepare_for_battle()
│   └─ _pre_tick_scan() → 全量重建 spatial hash + 队伍缓存
│
├─ ×400: unit.enter_combat()         # 第四次遍历
│   └─ play_anim_state(IDLE)
│
├─ _set_stage(Stage.COMBAT)          # 第五次遍历
│   ├─ _apply_visual_to_all_units()  ← 遍历 bench + ally + enemy
│   ├─ _refit_hex_grid()
│   ├─ _refresh_deployed_positions() ← 遍历 ally + enemy
│   ├─ _refresh_all_ui()             ← 重置 HUD timer + 全量刷新
│   ├─ _clear_battle_log()
│   ├─ _apply_stage_ui_state()
│   │   └─ _rebuild_inventory_items() ← 重建仓库面板
│   └─ signal → battlefield_ui._set_stage()
│
├─ _refresh_multimesh()              # 又一次 ally + enemy 遍历（多余）
├─ _refresh_all_ui()                 # 多余：_set_stage 已调用
│
└─ signal: battle_started
    └─ gongfa_manager._on_battle_started()
        └─ _fire_trigger_for_all("on_combat_start")  # 第六次 ×400
```

### 2.2 关键瓶颈量化

| 操作 | 单位数 | 单次成本 | 总成本估算 |
|------|--------|---------|-----------|
| `load("unit_data.gd")` ×400 | 400 | 脚本查找+缓存检查 | 显著 |
| `duplicate(true)` in `apply_gongfa` | 400×~5 | 递归深拷贝 | 2000 次分配 |
| `_apply_state_to_unit` ×400 | 400 | 4 层 effects + combat refresh | 极重 |
| `linkage.duplicate(true)` ×L×T | L×2 | 整个联动配置深拷贝 | L 个联动 × 2 队 |
| `_start_combat` 末尾多余的 `_refresh_multimesh` + `_refresh_all_ui` | 1 | ally+enemy 遍历 + UI 全刷 | 冗余 |

## 3. 持续卡顿根因（战斗中 FPS 逐降）

### 3.1 `_on_unit_died` 同帧爆发

每次死亡触发：
- `_remove_unit_from_map()` — O(n) 值扫描
- `_refresh_multimesh()` — 全量重建 MultiMesh
- `_refresh_all_ui()` — 重置 HUD timer + 全量 UI

批量击杀时同帧 5–10 次重复。

### 3.2 `_pre_tick_scan` 全量重建

每逻辑帧清空并重建 `_unit_by_instance_id` + `_spatial_hash` + 队伍缓存 + 流场格缓存。

### 3.3 非活跃单位空转

`unit_base._process()` 对所有单位（包括死亡/板凳）每帧调用 `movement.tick()`。

### 3.4 日志 flush 粒度

`_flush_battle_log()` 每次全量 `clear()` + 180 行 BBCode 重建。

## 4. 修复方案

### 第一批 —— 战斗开始卡顿（P0）

| # | 改动 | 文件 | 预期效果 |
|---|------|------|---------|
| 1 | `_start_combat()` 移除尾部多余的 `_refresh_multimesh()` + `_refresh_all_ui()` | `battlefield_runtime.gd` | 去掉 2 次冗余全量遍历 |
| 2 | `unit_base._apply_runtime_stats()` 缓存 `load("unit_data.gd")` 为 const | `unit_base.gd` | 消除 400 次 `load()` |
| 3 | `apply_gongfa()` 中 passive_effects / skill_data 改浅拷贝或引用 | `gongfa_manager.gd` | 减少 ~2000 次 `duplicate(true)` |
| 4 | `_refresh_linkages()` 只对有联动效果变化的单位调用 `_apply_state_to_unit` | `gongfa_manager.gd` | 400→有联动效果的单位数 |
| 5 | `linkage_detector._match_linkage()` 不对 linkage 做 `duplicate(true)` | `linkage_detector.gd` | 配置是只读的，不需要拷贝 |
| 6 | `vfx_factory._resolve_record()` 返回引用不做深拷贝 | `vfx_factory.gd` | 减少 VFX 热路径分配 |

### 第二批 —— 战斗中持续卡顿（P1）

| # | 改动 | 文件 | 预期效果 |
|---|------|------|---------|
| 7 | `_on_unit_died()` 改为 dirty flag 延迟合并刷新 | `battlefield_runtime.gd` | 每帧最多 1 次 multimesh 重建 |
| 8 | `_remove_unit_from_map()` 用 `deployed_cell` O(1) 移除 | `battlefield_runtime.gd` | O(n) → O(1) |
| 9 | `unit_base._process()` 对非移动单位 `set_process(false)` | `unit_base.gd` | ~450 → ~实际移动数 |
| 10 | `_flush_battle_log()` 增量 append，只在裁剪时全量重建 | `battlefield_ui.gd` | flush 成本 O(180) → O(增量) |

### 第三批 —— 精细优化（P2）

| # | 改动 | 文件 |
|---|------|------|
| 11 | `_pre_tick_scan()` 增量维护 `_unit_by_instance_id` | `combat_manager.gd` |
| 12 | `_handle_unit_death()` 精确计数代替 `_pre_tick_scan()` | `combat_manager.gd` |
| 13 | `_pick_visible_unit_at_world()` 改用空间哈希 | `battlefield_runtime.gd` |
| 14 | `queue_redraw()` 加 `_dragging_unit != null` 守卫 | `battlefield_runtime.gd` |
| 15 | ATTACK/HIT 动画改 `_process` 插值替代 Tween | `sprite_animator.gd` |

## 6. 验证方式

1. 布阵期空场 FPS 应稳定 ~60。
2. F6 开战后前 3 秒最低 FPS 不应掉到 30 以下。
3. 战斗持续 60 秒后 FPS 不应线性下滑。
4. 大量单位集中死亡时不应出现明显卡顿尖峰。
5. 战斗功能（攻击、死亡、VFX、结算）正常工作。
