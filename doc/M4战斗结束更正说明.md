# M4 战斗结束（暂停）流程更正说明

> 目标：一方全部阵亡后，**直接冻结棋盘上所有棋子**，不执行多余的清理/动画/状态重置。

---

## 一、当前流程问题分析

当一方存活数归零后，系统执行了**过多的额外处理**，导致结算瞬间出现不必要的视觉跳变和状态修改。

### 当前调用链

```
_handle_unit_death()          ← 最后一个单位阵亡
  └→ _finalize_if_needed()    ← 检测到某方 alive == 0
      └→ stop_battle("annihilation", winner)
          ├→ _battle_running = false
          ├→ [遍历 _all_units] _cleanup_unit_after_battle()   ← ⚠ 问题点
          │     ├ movement.clear_target()          清空移动目标
          │     ├ 判断胜负方 → play_anim_state(6)  播放胜利动画
          │     ├ 或 play_anim_state(0)            播放待机动画
          │     ├ unit.is_in_combat = false         修改战斗标记
          │     └ unit.leave_combat()              再次播放 IDLE + 刷新 process
          └→ battle_ended.emit(winner, summary)
              └→ _on_battle_ended()  (battlefield_runtime.gd)
                    ├ _set_stage(Stage.RESULT)
                    │   ├ _apply_visual_to_all_units()    全量视觉重刷（缩放/显隐切换）
                    │   ├ _refit_hex_grid()                重排棋盘
                    │   ├ _refresh_deployed_positions()     重定位所有棋子
                    │   └ _refresh_all_ui()                 刷新 HUD
                    └ _flush_pending_runtime_refreshes()
```

### 问题清单

| # | 位置 | 问题 |
|---|------|------|
| 1 | `_cleanup_unit_after_battle` | 遍历**所有**单位逐个调用 `clear_target()` + `leave_combat()` + 播放动画，400 单位时做 400 次 |
| 2 | `play_anim_state(6)` VICTORY | 胜利方棋子突然切换为胜利动画，干扰"冻结当前画面"的预期 |
| 3 | `play_anim_state(0)` IDLE | 败方存活棋子（如果有的话）被强制切回 IDLE |
| 4 | `leave_combat()` | 内部再次调用 `play_anim_state(0)` + `refresh_process_state()`，与上面重复 |
| 5 | `_set_stage(RESULT)` | 触发 `_apply_visual_to_all_units()` 全量重算单位缩放/显隐，画面闪跳 |
| 6 | `_refit_hex_grid()` + `_refresh_deployed_positions()` | 结算阶段重排棋盘布局 + 重定位所有棋子坐标，不必要 |

---

## 二、更正方案

### 核心原则

> **结算 = 暂停**。一方全灭后，所有棋子**原地冻结**，保持当前动画帧和位置不变。

### 2.1 修改 `combat_manager.gd` — `stop_battle()`

```diff
 func stop_battle(reason: String = "manual", winner_team: int = 0) -> void:
     if not _battle_running:
         return
     _battle_running = false
     var summary: Dictionary = _build_battle_summary(reason, winner_team)

-    for unit in _all_units:
-        if not _is_live_unit(unit):
-            continue
-        _cleanup_unit_after_battle(unit, winner_team)
+    # 冻结所有存活单位：只停止移动，不改动画/不改战斗标记
+    for unit in _all_units:
+        if not _is_live_unit(unit):
+            continue
+        var movement: Node = _get_movement(unit)
+        if movement != null:
+            movement.call("clear_target")

     battle_ended.emit(winner_team, summary)
```

**要点：**
- 只做 `clear_target()` 阻止棋子继续位移
- **不**调用 `play_anim_state()`，棋子保持当前动画帧（攻击/移动/受击等）
- **不**调用 `leave_combat()` / 修改 `is_in_combat`
- `_cleanup_unit_after_battle()` 函数保留但不在此处调用，可供后续"回到布阵期"时使用

### 2.2 修改 `battlefield_runtime.gd` — `_on_battle_ended()`

```diff
 func _on_battle_ended(winner_team: int, _summary: Dictionary) -> void:
-    _set_stage(Stage.RESULT)
-    _flush_pending_runtime_refreshes(true)
+    # 仅切换阶段标记和顶部 HUD，不触发全量视觉/布局重排
+    _stage = Stage.RESULT
+    phase_label.text = "结算期"
+    phase_label.modulate = Color(1.0, 0.86, 0.5, 1.0)
+    bench_ui.set_interactable(false)
+    deploy_overlay.visible = false
+    _set_shop_action_enabled(false)
+    _play_phase_transition("战斗结束")
+    # 刷新 HUD 数值（不重排棋盘布局）
+    _refresh_dynamic_ui()

     if winner_team == 1:
         debug_label.text = "战斗结束：己方胜利。F7 重开"
     elif winner_team == 2:
         debug_label.text = "战斗结束：敌方胜利。F7 重开"
     else:
         debug_label.text = "战斗结束：平局。F7 重开"
```

**要点：**
- **不调用** `_set_stage()`，因为它内部会触发 `_apply_visual_to_all_units()`, `_refit_hex_grid()`, `_refresh_deployed_positions()` 等全量重排
- 手动设置阶段相关的最小 UI 状态（标签文字、禁用操作按钮、隐藏部署区）
- **不调用 `_set_bottom_expanded()`**，底栏保持当前状态不弹出
- 只刷新 HUD 数值（存活数/计时器），不触发 multimesh/位置重算

### 2.3 `unit_base.gd` — `leave_combat()` 保持不变

`leave_combat()` 本身**不需要修改**，但需确保战斗结束时**不被调用**。该函数仅在从结算期回到布阵期时由上层主动调用。

---

## 三、涉及文件清单

| 文件 | 修改内容 |
|------|----------|
| `scripts/combat/combat_manager.gd` | `stop_battle()`: 移除 `_cleanup_unit_after_battle()` 循环，仅保留 `clear_target()` |
| `scripts/battle/battlefield_runtime.gd` | `_on_battle_ended()`: 不调用 `_set_stage()`，手动设最小 UI 状态 |

---

## 四、不需要修改的部分

| 组件 | 原因 |
|------|------|
| `_finalize_if_needed()` | 胜负判定逻辑正确，无需更改 |
| `_handle_unit_death()` | 阵亡处理（播放死亡动画、从存活列表移除）正确 |
| `_cleanup_unit_after_battle()` | 保留函数定义，后续"回到布阵期"时复用 |
| `unit_base.gd` / `leave_combat()` | 保留不变，不在战斗结束时调用即可 |
| 战斗统计面板相关代码 | 统计数据收集不受影响，面板展示可正常工作 |

---

## 五、预期行为对比

| 场景 | 修改前 | 修改后 |
|------|--------|--------|
| 最后一个敌方阵亡 | 所有己方→胜利动画，全量重排棋盘 | 所有棋子原地冻结，保持当前姿态 |
| 结算阶段底栏 | 底栏展开，触发 `_refit_hex_grid` | 底栏不变，棋盘布局不重排 |
| 棋子缩放/显隐 | `_apply_visual_to_all_units` 全量重算 | 不执行，保持战斗期的缩放比例 |
| 性能 | O(n) 遍历 + 多次动画切换 + 全量 UI 刷新 | 仅 O(n) 的 `clear_target()` + 最小 UI 更新 |

---

## 六、后续扩展点

如需在冻结后显示"胜利/失败"特效，可在 `_on_battle_ended()` 中延迟 1-2 秒后单独播放，而非在 `stop_battle()` 中与暂停逻辑耦合。例如：

```gdscript
# 可选：延迟播放胜利特效（不影响冻结效果）
await get_tree().create_timer(1.5).timeout
# 此处可选择性地对胜利方播放庆祝动画或 VFX
```
