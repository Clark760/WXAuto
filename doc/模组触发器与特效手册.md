# 模组触发器与特效手册

> 面向 Mod 作者。本文以当前代码为准，区分“单位触发管线”和“全局触发(EventBus)”。

## 1. 系统总览

- 单位触发（已实现）：走 `GongfaManager + TriggerEngine`，配置在技能 `trigger/trigger_params/effects`。
- 全局触发（已实现）：走 `EventBus.global_trigger_fired(trigger_name, payload)`，适合关卡阶段逻辑，不进入单位技能触发管线。
- 联动分支（已实现）：`effect.op = tag_linkage_branch`，支持 `query_type=match_tags/forbid_tags`、`exclude_tags`、`team_scope`、`execution_mode`。

最小技能结构：

```json
{
  "trigger": "on_attack_hit",
  "trigger_params": {},
  "cooldown": 4,
  "mp_cost": 20,
  "effects": [
    {"op":"damage_target","value":120}
  ]
}
```

### 1.1 Buff 系统（Mod 作者必读）

Buff 由 `buffs/*.json` 定义，运行时由 `BuffManager` 管理生命周期，`GongfaManager` 在每次重算属性时收集其被动效果。

Buff 定义字段（`data/buffs/_schema/buff.schema.json`）：

| 字段 | 必填 | 类型 | 用途 | 建议 |
|---|---|---|---|---|
| `id` | 是 | string | Buff 唯一ID | 小写+下划线，稳定不改名。 |
| `name` | 是 | string | 显示名称 | 建议人类可读。 |
| `type` | 是 | `buff/debuff` | 正面/负面类型 | `debuff` 会受韧性缩短时长。 |
| `icon` | 否 | string | 图标路径 | 可空。 |
| `stackable` | 是 | bool | 是否可叠层 | 可叠层时配 `max_stacks>1`。 |
| `max_stacks` | 是 | int | 最大层数 | 建议 1~10。 |
| `default_duration` | 否 | number | 默认持续时间(秒) | effect 不传 duration 时使用。 |
| `effects` | 是 | array | 被动效果列表 | 每次属性重算生效。 |
| `tick_effects` | 是 | array | 周期触发效果列表 | 按 `tick_interval` 执行。 |
| `tick_interval` | 是 | number | tick 间隔(秒) | `0` 表示不tick。 |

Buff 常用施加入口（Effect）：

| op | 关键参数 | 说明 |
|---|---|---|
| `buff_self` | `buff_id`,`duration` | 给自己上 Buff。 |
| `buff_target` | `buff_id`,`duration` | 给目标上 Buff。 |
| `buff_allies_aoe` | `buff_id`,`duration`,`radius`,`exclude_self` | 给范围友方上 Buff。 |
| `debuff_target` | `buff_id`,`duration` | 给目标上 Debuff。 |
| `debuff_aoe` | `buff_id`,`duration`,`radius` | 给范围敌方上 Debuff。 |
| `shield_self` | `shield_buff_id/buff_id`,`duration` | 护盾+绑定Buff。 |
| `shield_allies_aoe` | `shield_buff_id`,`duration`,`radius`,`exclude_self` | 群体护盾Buff。 |

运行规则（当前代码行为）：

1. 叠层键是 `buff_id + source_id`。同来源重复施加会叠层/刷新；不同来源会并存为独立条目。
2. `stackable=true` 时重复施加会加层，直到 `max_stacks`；否则只刷新持续时间。
3. `duration < 0` 会被视为永久（仅在移除/死亡/驱散时结束）。
4. `binding_mode = source_bound_aura` 仅对 `buff_allies_aoe` / `debuff_aoe` 生效：运行时按“来源绑定光环”处理，Buff 实例会以永久时长挂载，但会在提供者死亡或目标脱离光环条件时自动移除。
5. Debuff 且持续时间大于 0 时会按目标 `tenacity` 缩短。
6. 叠层放大仅自动作用在 `effects` 中带 `value` 的条目（线性乘层数）。
7. `tick_effects` 当前不会按 `stacks` 自动放大；如果你要“层数越高每跳越痛”，建议拆为多来源或改为被动增伤链路。

最小 Buff 定义示例：

```json
{
  "id": "buff_haste_basic",
  "name": "迅疾",
  "type": "buff",
  "stackable": true,
  "max_stacks": 3,
  "default_duration": 4.0,
  "effects": [
    {"op":"attack_speed_bonus","value":0.06}
  ],
  "tick_effects": [],
  "tick_interval": 0.0
}
```

## 2. Trigger 总表（Mod写法）

### 2.1 轮询触发（单位）

| Trigger | 状态 | 用途 | trigger_params | 最小示例 | 注意事项 |
|---|---|---|---|---|---|
| `auto_mp_full` | 已实现 | 满蓝自动释放 | 无 | `{"trigger":"auto_mp_full"}` | 仍受 `mp_cost/cooldown/max_trigger_count` 约束。 |
| `manual` | 已实现 | 手动释放/通用门控 | 无 | `{"trigger":"manual"}` | 主要用于UI主动施法。 |
| `auto_hp_below` | 已实现 | 低血自动释放 | 读取 `skill.threshold` | `{"trigger":"auto_hp_below","threshold":0.35}` | 不是边沿触发，建议配冷却。 |
| `on_hp_below` | 已实现 | 首次跌破血线触发 | `threshold` | `{"trigger":"on_hp_below","trigger_params":{"threshold":0.25}}` | 只在“上->下”边沿触发。 |
| `on_time_elapsed` | 已实现 | 战斗达到时点触发 | `at_seconds` | `{"trigger":"on_time_elapsed","trigger_params":{"at_seconds":8}}` | 默认一次性。 |
| `periodic_seconds` | 已实现 | 周期触发 | `interval` | `{"trigger":"periodic_seconds","trigger_params":{"interval":0.8}}` | 高频会增加计算量。 |
| `passive_aura` | 已实现 | 轮询型光环 | 无 | `{"trigger":"passive_aura"}` | 与 `tag_linkage_branch stateful` 搭配更稳。 |

### 2.3 全局触发（EventBus）

订阅方式：

```gdscript
var event_bus := get_tree().root.get_node_or_null("EventBus")
if event_bus != null:
    event_bus.global_trigger_fired.connect(_on_global_trigger)

func _on_global_trigger(trigger_name: String, payload: Dictionary) -> void:
    if trigger_name == "on_stage_completed":
        print(payload)
```

| Trigger | 状态 | 触发时机 | payload关键字段 |
|---|---|---|---|
| `on_preparation_started` | 已实现 | 进入关卡准备阶段 | `stage_id`, `config` |
| `on_stage_combat_started` | 已实现 | 关卡正式开战 | `stage_id`, `config` |
| `on_stage_completed` | 已实现 | 关卡胜利结算 | `stage_id`, `config`, `rewards`, `winner_team`, `without_battle` |
| `on_stage_failed` | 已实现 | 关卡失败结算 | `stage_id`, `config`, `winner_team` |
| `on_stage_loaded` | 规划 | 仅加载完成时回调 | 复用 `config` |
| `on_all_stages_cleared` | 规划 | 序列全通时 | 可扩展 |

## 5. Tag 联动写法（`tag_linkage_branch`）

核心字段：

- `queries[]`：查询定义
- `query_type`: `match_tags`（正向）或 `forbid_tags`（负向）
- `tags/tag_match`：正向命中
- `exclude_tags/exclude_match`：排除或负向匹配
- `count_mode`: `provider/unit`
- `cases[]`：阈值分支（`min_count/max_count`）
- `execution_mode`: `continuous/stateful`

### 示例A：五格内阵型 3/5/7 档

```json
{
  "op": "tag_linkage_branch",
  "execution_mode": "stateful",
  "range": 5,
  "team_scope": "ally",
  "count_mode": "provider",
  "queries": [
    {
      "id": "zhenyan",
      "query_type": "match_tags",
      "source_types": ["trait","gongfa","equipment"],
      "tags": ["tag_true_martial_array"],
      "tag_match": "any"
    }
  ],
  "cases": [
    {"id":"seven","min_count":7,"effects":[{"op":"damage_amp_percent","value":0.35}]},
    {"id":"five","min_count":5,"max_count":6,"effects":[{"op":"damage_amp_percent","value":0.2}]},
    {"id":"three","min_count":3,"max_count":4,"effects":[{"op":"damage_amp_percent","value":0.1}]}
  ],
  "else_effects": []
}
```

### 示例B：有A且无B（`match + exclude`）

## 6. 常见坑与性能建议

- 高频触发（`periodic_seconds/on_unit_move_success/on_terrain_tick`）必须配冷却、阈值或 `reasons` 过滤。
- `tag_linkage_branch` 优先用 `stateful`，避免每次都重复刷短Buff。
- “全图条件”优先用事件与计数条件（如 `team_alive_count_*`），不要用超大范围轮询扫描。
- 复杂连锁建议拆两层：
  - 第一层 trigger 做门控（时机、原因、阈值）
  - 第二层 effect 做结果（控制、伤害、召唤、地形）
- 全局触发不要混入单位技能触发管线；阶段逻辑统一订阅 `EventBus.global_trigger_fired`。

说明：
- 这是“完整光环”写法：命中的目标会拿到一个运行时永久 Buff，但生命周期不看 `duration`，而看光环条件。

  "max_stacks": 8,
