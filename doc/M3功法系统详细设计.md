# M3 功法系统详细设计（面向 AI 代码生成）

> M3 是本游戏的**核心差异化系统**，取代传统自走棋的羁绊机制。
> 本文档定义功法效果引擎、联动检测器、数据结构和示例 JSON。所有数据均为外部 JSON，支持 Mod 覆盖/追加。

---

## 二、功法数据结构

### 2.1 功法 JSON Schema

路径：`data/gongfa/*.json`，每个文件可包含一个或多个功法。

```json
{
  "id": "gongfa_jiuyang",
  "name": "九阳神功",
  "type": "neigong",
  "description": "至阳至刚的内功心法，大幅提升生命与内力恢复",
  "faction": "shaolin",
  "element": "fire",
  "quality": "orange",
  "tags": ["yang", "recovery", "shaolin_core"],

  "passive_effects": [
    {"op": "stat_add", "stat": "hp", "value": 200},
    {"op": "stat_add", "stat": "mp", "value": 80},
    {"op": "stat_percent", "stat": "hp", "value": 0.15},
    {"op": "mp_regen_add", "value": 5.0}
  ],

  "skill": {
    "trigger": "auto_mp_full",
    "mp_cost": 60,
    "cooldown": 0,
    "effects": [
      {"op": "heal_self_percent", "value": 0.25},
      {"op": "buff_self", "buff_id": "jiuyang_shield", "duration": 5.0}
    ],
    "vfx_id": "vfx_golden_aura"
  },

  "linkage_tags": ["jiuyang", "yang_gong", "shaolin_neigong"]
}
```

### 2.2 type 枚举

| type 值 | 中文 | 功法行为 |
|---------|------|---------|
| `neigong` | 内功心法 | 被动属性加成 + 可选主动技能 |
| `waigong` | 外功招式 | 主动技能为主（替换/增强普攻） |
| `qinggong` | 轻功身法 | 移动相关技能（闪避、位移、加速） |
| `zhenfa` | 阵法 | 部署型区域增益 |
| `qishu` | 奇术 | 特殊效果（复活、变身、嘲讽等） |

### 2.3 element 五行

| element | 中文 | 相生（增益） | 相克（伤害加成） |
|---------|------|------------|----------------|
| `metal` | 金 | 生水 | 克木 |
| `wood` | 木 | 生火 | 克土 |
| `water` | 水 | 生木 | 克火 |
| `fire` | 火 | 生土 | 克金 |
| `earth` | 土 | 生金 | 克水 |
| `none` | 无 | 无特殊 | 无特殊 |

## 三、效果引擎（EffectEngine）

### 3.1 效果操作（EffectOp）定义

所有 `passive_effects` 和 `skill.effects` 中的条目都是一个 `EffectOp` 字典。EffectEngine 按 `op` 字段分派处理。

#### 被动效果 Op（装备功法时立即生效）

| op | 参数 | 说明 |
|----|------|------|
| `stat_add` | `stat`, `value` | 属性平加：`runtime_stats[stat] += value` |
| `stat_percent` | `stat`, `value` | 属性百分比加成：`runtime_stats[stat] *= (1 + value)` |
| `mp_regen_add` | `value` | 内力每秒回复增加 |
| `damage_reduce_flat` | `value` | 受到伤害固定减免 |
| `damage_reduce_percent` | `value` | 受到伤害百分比减免 |
| `dodge_bonus` | `value` | 闪避率加成（加到公式上） |
| `crit_bonus` | `value` | 暴击率加成 |
| `crit_damage_bonus` | `value` | 暴击伤害倍率加成 |
| `attack_speed_bonus` | `value` | 攻击间隔缩短百分比 |
| `range_add` | `value` | 攻击范围加格 |

#### 主动效果 Op（技能触发时执行）

| op | 参数 | 说明 |
|----|------|------|
| `damage_target` | `value`, `damage_type`, `multiplier` | 对当前攻击目标造成伤害 |
| `damage_aoe` | `radius`, `value`, `damage_type` | 对自身周围范围造成伤害 |
| `heal_self` | `value` | 自身回复固定 HP |
| `heal_self_percent` | `value` | 自身回复最大 HP 百分比 |
| `heal_allies_aoe` | `radius`, `value` | 范围内友方回复 |
| `buff_self` | `buff_id`, `duration` | 给自身施加 Buff |
| `buff_allies_aoe` | `buff_id`, `radius`, `duration` | 范围内友方施加 Buff |
| `debuff_target` | `buff_id`, `duration` | 给攻击目标施加 Debuff |
| `debuff_aoe` | `buff_id`, `radius`, `duration` | 范围内敌方施加 Debuff |
| `teleport_behind` | `distance` | 闪现到目标背后 |
| `dash_forward` | `distance` | 向目标方向冲刺 |
| `knockback_target` | `distance` | 击退目标 |
| `summon_clone` | `unit_id`, `count`, `duration` | 召唤临时分身 |
| `revive_random_ally` | `hp_percent` | 复活一个随机死亡友方 |
| `taunt_aoe` | `radius`, `duration` | 范围嘲讽，强制敌人攻击自己 |
| `spawn_vfx` | `vfx_id`, `at` | 播放特效（`at` = `self`/`target`/`aoe_center`） |

### 3.2 Buff/Debuff 数据结构

路径：`data/buffs/*.json`

```json
{
  "id": "jiuyang_shield",
  "name": "九阳护体",
  "type": "buff",
  "icon": "buff_golden_shield",
  "stackable": false,
  "max_stacks": 1,
  "effects": [
    {"op": "damage_reduce_percent", "value": 0.30},
    {"op": "stat_percent", "stat": "def", "value": 0.20}
  ],
  "tick_effects": [],
  "tick_interval": 0
}
```

DOT 型 Debuff 示例：

```json
{
  "id": "tangmen_poison",
  "name": "唐门剧毒",
  "type": "debuff",
  "icon": "debuff_poison",
  "stackable": true,
  "max_stacks": 5,
  "effects": [
    {"op": "stat_percent", "stat": "spd", "value": -0.15}
  ],
  "tick_effects": [
    {"op": "damage_target", "value": 30, "damage_type": "internal"}
  ],
  "tick_interval": 1.0
}
```

### 3.3 技能触发器（Trigger）

`skill.trigger` 定义技能何时自动施放：

| trigger 值 | 说明 |
|------------|------|
| `auto_mp_full` | 内力满时自动释放（消耗 `mp_cost`） |
| `auto_hp_below` | HP 低于某比例时触发（附 `threshold: 0.3`） |
| `on_attack_hit` | 普攻命中时概率触发（附 `chance: 0.25`） |
| `on_attacked` | 被攻击时概率触发（附 `chance: 0.20`） |
| `on_kill` | 击杀敌人时触发 |
| `on_ally_death` | 友方死亡时触发 |
| `on_combat_start` | 战斗开始时立即触发一次 |
| `passive_aura` | 持续光环效果，无需触发 |
| `manual` | 预留，当前版本等同 `auto_mp_full` |

## 四、联动检测器（LinkageDetector）

### 4.1 联动数据结构

路径：`data/linkages/*.json`

```json
{
  "id": "linkage_sancai_jian",
  "name": "三才剑阵",
  "type": "faction_combo",
  "description": "3名武当弟子修炼全真剑法时，攻击力+30%，格挡+15%",
  "icon": "linkage_wudang_sword",

  "conditions": {
    "min_count": 3,
    "require_faction": "wudang",
    "require_any_tag": ["quanzhen_sword"],
    "require_adjacent": true
  },

  "effects": [
    {"op": "stat_percent", "stat": "atk", "value": 0.30, "target": "participants"},
    {"op": "dodge_bonus", "value": 0.15, "target": "participants"}
  ],

  "vfx_id": "vfx_sword_formation"
}
```

### 4.2 联动类型

| type 值 | 中文 | conditions 必填字段 |
|---------|------|-------------------|
| `faction_combo` | 同门联动 | `require_faction`, `min_count`, 可选 `require_any_tag` |
| `element_resonance` | 五行共鸣 | `require_elements`（数组，五行对） |
| `skill_chain` | 武技连携 | `require_tags_combo`（数组，需多角色各持有一个 tag） |
| `formation_boost` | 阵法联动 | `require_zhenfa_tag`, `require_any_tag`（增益对象的 tag） |

### 4.3 conditions 字段详解

| 字段 | 类型 | 说明 |
|------|------|------|
| `min_count` | int | 满足条件的最少角色数 |
| `require_faction` | string | 要求角色所属势力 |
| `require_any_tag` | string[] | 角色功法的 `linkage_tags` 中至少包含一个 |
| `require_all_tags` | string[] | 角色功法的 `linkage_tags` 必须全部包含 |
| `require_adjacent` | bool | 参与者必须站位相邻（六角格相邻） |
| `require_elements` | [string, string][] | 五行对列表，检查场上是否存在该五行组合 |
| `require_tags_combo` | string[][] | 多组 tag，每组由不同角色满足 |
| `require_zhenfa_tag` | string | 需要场上有某阵法功法激活 |
| `max_trigger_count` | int | 每场战斗最多触发次数（默认无限） |

### 4.4 联动效果 target

| target 值 | 说明 |
|-----------|------|
| `participants` | 满足条件的参与者本身 |
| `all_allies` | 全部己方存活单位 |
| `zhenfa_area` | 阵法范围内的友方 |
| `random_enemy` | 随机一个敌方 |

### 4.5 检测时机

```gdscript
# LinkageDetector 在以下时机执行检测：
# 1. 布阵期：每次部署/移动/移除棋子后
# 2. 交锋期：战斗开始时执行一次完整检测
# 3. 交锋期：每 N 逻辑帧（如 10 帧=1秒）刷新一次（检测阵法范围变化）

func detect_all(deployed_units: Array[Node]) -> Array[LinkageResult]:
    var results: Array[LinkageResult] = []
    for linkage_data in _all_linkages:
        var matched = _check_conditions(linkage_data, deployed_units)
        if matched.size() > 0:
            results.append(LinkageResult.new(linkage_data, matched))
    return results
```

### 4.6 LinkageResult 结构

```gdscript
class LinkageResult:
    var linkage_id: String           # 联动 ID
    var linkage_data: Dictionary     # 联动原始数据
    var participants: Array[Node]    # 参与角色列表
    var is_active: bool              # 当前是否激活
```

## 五、角色功法槽位

每个 UnitBase 拥有功法槽位：

```json
// 角色 JSON 中的功法配置
{
  "id": "unit_zhang_sanfeng",
  "gongfa_slots": {
    "neigong": "gongfa_taiji_neigong",
    "waigong": "gongfa_taijiquan",
    "qinggong": null,
    "zhenfa": null,
    "qishu": null
  },
  "max_gongfa_count": 3
}
```

| 槽位规则 | 说明 |
|---------|------|
| 每类最多装 1 个 | 内功/外功/轻功/阵法/奇术各最多 1 个 |
| 总数上限 | 由 `max_gongfa_count` 控制，1-3 星角色通常为 2-3 |
| 更换 | 布阵期可自由更换，交锋期锁定 |

### 运行时功法系统流程

```
角色加入战场
    │
    ▼
GongfaManager.apply_gongfa(unit)
    │
    ├─ 读取 unit.gongfa_slots
    ├─ 对每个功法 ID → 从 GongfaRegistry 取数据
    ├─ 执行 passive_effects → 修改 unit.runtime_stats
    ├─ 注册 skill.trigger → 挂载到 CombatManager 事件
    └─ 收集 linkage_tags → 存入 unit 供 LinkageDetector 查询
    │
    ▼
LinkageDetector.detect_all(all_deployed_units)
    │
    ├─ 遍历所有联动数据
    ├─ 检查 conditions 是否满足
    ├─ 满足 → 对 target 应用联动 effects
    └─ 返回激活的联动列表 → 更新 UI
```

## 六、GongfaManager 接口

```gdscript
# scripts/gongfa/gongfa_manager.gd

# 为角色应用所有已装备功法的被动效果
func apply_gongfa(unit: Node) -> void

# 移除角色所有功法效果（角色离场/换功法时）
func remove_gongfa(unit: Node) -> void

# 为角色装备新功法（布阵期操作）
func equip_gongfa(unit: Node, slot: String, gongfa_id: String) -> bool

# 卸下角色某槽位功法
func unequip_gongfa(unit: Node, slot: String) -> void

# 获取功法数据
func get_gongfa_data(gongfa_id: String) -> Dictionary

# 获取所有功法列表（含 Mod 加载的）
func get_all_gongfa() -> Array[Dictionary]
```
