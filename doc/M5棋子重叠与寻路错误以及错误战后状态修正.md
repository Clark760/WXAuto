## 修改一：后方单位来回徘徊（寻路震荡）

### 根因

`combat_manager.gd` 行 36：`block_teammate_cells_in_flow = false`

流场把友方占格视为可通行并分配了 cost，但实际移动时 `_is_cell_free()` 阻挡友方占格。后排单位看到的所有可走空格 flow cost ≥ 当前格 → 无下坡格 → `allow_equal_cost_side_step` 和 `allow_uphill_escape_step` 驱动横移/上坡 → 下帧回来 → 震荡。

### 修改内容

#### 修改 1.1：`combat_manager.gd` 行 36

```diff
-@export var block_teammate_cells_in_flow: bool = false
+@export var block_teammate_cells_in_flow: bool = true
```

#### 修改 1.2：`combat_pathfinding.gd` 行 127-139

**禁用上坡逃逸步**（启用 `block_teammate_cells_in_flow` 后流场已正确绕路，不再需要上坡兜底）：

```diff
-	if bool(owner.get("allow_uphill_escape_step")) and neighbor_cost > current_cost:
-		if not uphill_found or neighbor_cost < uphill_cost or (neighbor_cost == uphill_cost and neighbor_focus_dist < uphill_focus_dist):
-			uphill_found = true
-			uphill_cost = neighbor_cost
-			uphill_focus_dist = neighbor_focus_dist
-			uphill_cell = neighbor
```

改为：

```gdscript
	# M5-FIX: 禁用上坡逃逸步。block_teammate_cells_in_flow=true 后
	# 流场已正确绕路，上坡步只会导致后排单位远离目标再靠近的震荡。
	# 保留变量但不再赋值。
```

同时修改行 138：

```diff
-	if bool(owner.get("allow_uphill_escape_step")) and uphill_found and current_cost > 3:
-		return uphill_cell
+	# M5-FIX: 上坡逃逸步已禁用
```

> [!NOTE]
> 也可以直接在 `combat_manager.gd` 行 37 改 `allow_uphill_escape_step = false`（如果该行存在）。效果等价但更简单。

## 修改二：两个单位占用同一格

### 根因

修改一（`block_teammate_cells_in_flow=true`）会**大幅减少**重叠问题，因为流场不再引导多个单位穿越同一友方占格。但仍有一个次发路径：

`cell_occupancy.get_unit_cell()` 只读版本（行 83-93）在 `_unit_cell` 缺失时返回 `(-1,-1)`。此时 `_run_unit_logic` 会跳过移动（行 574-581）。但 `_pre_tick_scan` 中的 `resolve_and_register_unit_cell()`（行 96-115）在 Tween 中途可能把单位注册到中间格。

### 修改内容

#### 修改 2.1：`cell_occupancy.gd` — `resolve_and_register_unit_cell` 行 105-111

在占格前增加额外的 Tween 状态检查：

```diff
 	var resolved_cell: Vector2i = fallback_cell
 	if not is_cell_free(owner, resolved_cell):
 		resolved_cell = find_nearest_free_cell(owner, fallback_cell)
 	if resolved_cell.x < 0:
 		return Vector2i(-1, -1)
+	# M5-FIX: 如果 resolved_cell 与 fallback_cell 不同，说明单位的 world position
+	# 对应的格已被占（可能是 Tween 中途）。此时不强制注册到邻格，
+	# 避免逻辑格与视觉位置不一致导致重叠。
+	if resolved_cell != fallback_cell:
+		return Vector2i(-1, -1)
 	if not occupy_cell(owner, resolved_cell, unit):
 		return Vector2i(-1, -1)
```

#### 修改 2.2：`combat_manager.gd` — `_run_unit_logic` 中移动提交后强制吸附位置

确认行 616-648 中在 `_occupy_cell` 成功后，所有分支都会更新 `unit.position` 到 `target_world`。当前代码已做到（行 627-647），无需额外修改。但为防 Tween 残留，在战斗结束清理中强制吸附：

确认 `stop_battle` 函数中是否调用了 `_cleanup_unit_after_battle`：

```
（需 AI 确认 stop_battle 中对每个存活单位调用 _cleanup_unit_after_battle 时,
  是否已有 position 吸附逻辑。如果没有，需要在 _cleanup_unit_after_battle
  开头添加以下代码：）
```

```gdscript
# _cleanup_unit_after_battle 开头添加：
# 杀掉格子移动 Tween，强制吸附到逻辑格心
if unit.has_method("kill_quick_step_tween"):
```
