# M5 效果系统扩充 & 全局联动/羁绊删除（面向代码 AI）

> **目标读者**：AI 代码生成器。
> **前置依赖**：M0–M4 全部完成，M5 关卡系统已实现。
> **核心变更**：① 大幅扩充 EffectOp / Buff / Debuff / 装备触发器 / 功法技能种类；② 删除全局联动（羁绊）系统及相关 UI。

---

## 一、全局联动系统删除

### 1.1 删除原因

当前联动系统是"全图判断"模式——只要场上满足条件（如 3 名少林弟子）就激活增益。在 50v200+ 的大战场中：
- 玩家上 50 人，几乎必然凑满所有门派联动，**无策略抉择**
- 全图遍历联动条件浪费性能
- UI 联动面板在大战场中遮挡视野且信息过载

### 1.2 需要删除的代码/文件

#### 代码文件

| 文件 | 操作 | 说明 |
|------|------|------|
| `scripts/gongfa/linkage_detector.gd` | **整文件删除** | 联动检测器核心 |
| `scripts/gongfa/gongfa_manager.gd` | **修改** | 移除所有 `_refresh_linkages` / `linkage_effects` / `LinkageResult` 相关逻辑 |
| `scripts/gongfa/gongfa_registry.gd` | **修改** | 移除 `_linkage_map` 及相关加载/查询方法 |
| `scripts/data/data_manager.gd` | **修改** | 移除 `linkages` 数据类别的加载 |
| `scripts/board/battlefield.gd` | **修改** | 移除 `_prepare_linkage_panel` / `_sync_linkage_panel_visibility` / `_get_linkage_panel_control` 及所有调用处（约行 143-144, 228, 1668, 2116-2131, 2211+） |

#### UI 场景

| 节点 | 操作 |
|------|------|

#### GongfaManager 修改详情

在 `_apply_state_to_unit` 中删除第 3 层联动效果：

```gdscript
# 修改前（4 层叠加）：
# 第1层：功法被动
# 第2层：装备被动
# 第3层：联动效果 ← 删除此层
# 第4层：Buff 效果

# 修改后（3 层叠加）：
# 第1层：功法被动
# 第2层：装备被动
# 第3层：Buff 效果
```

在 `apply_gongfa` 中删除：
- `linkage_tags` 收集逻辑
- `runtime_linkage_tags` / `runtime_elements` 字段
- `linkage_effects` 数组
- `_refresh_linkages()` 调用

在 `_unit_states` 字典中删除：
- `"linkage_effects"` 键
- `"runtime_linkage_tags"` 键
- `"runtime_elements"` 键

### 1.3 替代方案

联动的策略深度由以下机制替代（详见后续章节）：
- **装备触发器**：带有条件联动的装备特效（如"目标中毒时伤害 +50%"）
- **功法 Buff/Debuff 协同**：状态栏联动机制（A 施毒 → B 的技能对中毒目标额外伤害）
- **地形互动**：功法创建地形 + 地形增益/减益
- **阵法功法**：保留阵法类功法的局部区域增益（这是合理的局部联动）

## 二、新增 EffectOp（效果操作）

### 2.1 新增被动效果 Op

在 `EffectEngine` 中新增以下被动 Op：

| op | 参数 | 说明 |
|----|------|------|
| `hp_regen` | `value` | 每秒生命回复（已有部分装备使用，需确保引擎支持） |
| `vampire` | `value` | 生命偷取比例（普攻伤害 × value 回复 HP） |
| `damage_amp_percent` | `value` | 造成伤害增幅百分比（最终伤害 × (1 + value)） |
| `damage_amp_vs_debuffed` | `value`, `require_debuff`(可选) | 对身上有 Debuff 的目标造成额外伤害。`require_debuff` 指定特定 debuff_id，不填则任意 debuff 均可 |
| `crit_damage_bonus` | `value` | 暴击伤害倍率加成（已有，确认支持） |
| `tenacity` | `value` | 韧性：减少受控时间百分比 |
| `thorns_percent` | `value` | 反伤比例：受到物理伤害时反弹 value% |
| `thorns_flat` | `value` | 固定反伤：受到攻击时反弹固定伤害 |
| `shield_on_combat_start` | `value` | 战斗开始时获得 value 点护盾 |
| `execute_threshold` | `value` | 斩杀线：目标 HP 低于 value% 时造成双倍伤害 |
| `healing_amp` | `value` | 治疗效果增强百分比 |
| `mp_on_kill` | `value` | 击杀回复内力 |
| `conditional_stat` | `stat`, `value`, `condition`, `threshold` | 条件属性加成（如 HP 低于 50% 时 ATK +30%）。`condition` 枚举：`hp_below`/`hp_above`/`mp_below`/`mp_above` |

### 2.2 新增主动效果 Op

| op | 参数 | 说明 |
|----|------|------|
| `damage_target_scaling` | `value`, `damage_type`, `scale_stat`, `scale_ratio` | 基于某属性缩放的伤害（如基于最大 HP 的 5% 造成伤害）。`scale_stat` 可选 `max_hp`/`current_hp`/`atk`/`iat`/`def`/`idr` |
| `damage_if_debuffed` | `value`, `damage_type`, `require_debuff`, `bonus_multiplier` | 若目标身上有指定 debuff，伤害 × bonus_multiplier |
| `damage_chain` | `value`, `damage_type`, `chain_count`, `decay` | 链式伤害：命中后弹射到 chain_count 个邻近敌人，每次伤害衰减 decay% |
| `damage_cone` | `value`, `damage_type`, `angle`, `range` | 扇形伤害：以面朝方向为轴，angle 度锥形范围 |
| `heal_lowest_ally` | `value` | 治疗血量最低的友方 |
| `heal_percent_missing_hp` | `value` | 治疗目标已损失 HP 的 value%（低血量越有效） |
| `shield_self` | `value`, `duration` | 给自身施加 value 点护盾，持续 duration 秒 |
| `shield_allies_aoe` | `value`, `duration`, `radius` | 范围内友方获得护盾 |
| `cleanse_self` | — | 移除自身所有 Debuff |
| `cleanse_ally` | — | 移除血量最低友方的所有 Debuff |
| `steal_buff` | `count` | 偷取目标身上 count 个 Buff 转移给自己 |
| `dispel_target` | `count` | 驱散目标身上 count 个 Buff |
| `pull_target` | `distance` | 将目标拉向自己 distance 格 |
| `knockback_aoe` | `radius`, `distance` | 范围击退周围所有敌人 |
| `swap_position` | — | 与目标交换位置 |
| `create_terrain` | `terrain_type`, `radius`, `duration` | 在目标点创建临时地形（详见§五） |
| `mark_target` | `mark_id`, `duration` | 标记目标（可与 `damage_if_marked` 联动） |
| `damage_if_marked` | `value`, `damage_type`, `mark_id`, `bonus_multiplier` | 若目标有指定标记，伤害加成 |
| `execute_target` | `hp_threshold`, `damage_type`, `value` | 目标 HP 低于 threshold% 时立即造成 value 点处决伤害 |
| `drain_mp` | `value` | 吸取目标 value 点内力，转移给自己 |
| `silence_target` | `duration` | 沉默目标，duration 秒内无法释放技能 |
| `stun_target` | `duration` | 定身目标（移动+攻击全部停止） |
| `fear_aoe` | `radius`, `duration` | 范围恐惧：敌人向反方向逃跑 |
| `burn_ground` | `radius`, `damage_per_second`, `duration` | 在当前位置创建灼烧地面 |
| `freeze_target` | `duration` | 冰冻目标：完全定身 + 受到的下一次攻击暴击 |
| `resurrect_self` | `hp_percent`, `cooldown` | 死亡时自动复活（每场战斗限一次），回复 hp_percent% 生命 |
| `aoe_percent_hp_damage` | `radius`, `percent`, `damage_type` | 范围内敌人受到最大 HP percent% 的伤害 |

### 2.3 新增 Trigger（技能触发器）

| trigger 值 | 参数 | 说明 |
|------------|------|------|
| `on_crit` | `chance` | 暴击时触发 |
| `on_dodge` | `chance` | 闪避成功时触发 |
| `on_hp_below` | `threshold`, `cooldown` | HP 低于阈值时触发（带冷却避免反复触发） |
| `on_debuff_applied` | `chance` | 被施加 Debuff 时触发 |
| `on_buff_expire` | `buff_id` | 指定 Buff 到期时触发 |
| `periodic` | `interval` | 每 interval 秒自动触发一次 |

## 四、装备系统扩充

### 4.1 设计理念

现有 35 件装备均为纯属性加成（stat_add / stat_percent），没有触发器。扩充后装备分为三个层次：

| 层次 | 品质 | 特点 |
|------|------|------|
| **基础装备** | 白/绿 | 纯属性加成 |
| **进阶装备** | 蓝/紫 | 属性 + 简单触发器（如攻击附毒） |
| **神兵利器** | 橙/红 | 属性 + 复杂触发器 + 状态联动效果 |

## 五、临时地形系统（`create_terrain` Op 实现）

### 5.1 概述

`create_terrain` 是新增的主动效果 Op，功法/装备触发器可在目标点创建**临时地形区域**，类似 LOL 的技能地面效果。临时地形与关卡障碍物(`ObstacleManager`)共存但独立管理。

### 5.2 临时地形类型

| terrain_type | 名称 | 效果 | VFX |
|-------------|------|------|-----|
| `fire` | 火域 | 区域内敌人每秒受火焰伤害 | 地面橙红色粒子 |
| `ice` | 冰域 | 区域内敌人移速 -50%，攻速 -30% | 地面冰蓝色闪光 |
| `poison` | 毒沼 | 区域内敌人叠加中毒 debuff | 地面紫绿色冒泡 |
| `heal` | 灵泉 | 区域内友方每秒回复 HP | 地面绿色光柱 |
| `barrier` | 气墙 | 不可穿越的临时障碍（仅阻挡移动） | 半透明力场 |
| `slow` | 泥泞 | 移速 -70% | 地面棕色粒子 |
| `amp_damage` | 杀阵 | 区域内友方伤害 +20% | 地面红色法阵光环 |

### 5.3 数据结构

```json
{
  "op": "create_terrain",
  "terrain_type": "fire",
  "radius": 3,
  "duration": 8.0,
  "damage_per_second": 25,
  "at": "target"
}
```

### 5.4 实现要点

1. 新增 `scripts/board/terrain_manager.gd` — 管理临时地形生命周期
2. 临时地形在 `HexGrid` 中以 `terrain_effects` 数组标记（不同于 `blocked`）
3. `CombatManager` 每逻辑帧检查角色所在格是否有地形效果 → 施加对应 Buff/Debuff/伤害
4. 地形到期后自动清除
5. `barrier` 类型需要通知 `FlowField` 重建

## 七、状态栏联动机制说明

### 7.1 核心理念

取代全局羁绊的新策略深度来源是**状态栏联动**——角色 A 施加 Debuff，角色 B 的功法/装备对带有该 Debuff 的目标有加深效果：

```
角色A（唐门暗器师）攻击 → 目标中毒（tangmen_poison）
                                    ↓
角色B（五毒手功法）检测到目标有 tangmen_poison
                                    ↓
        damage_amp_vs_debuffed +25% → 伤害大幅提升
        damage_if_debuffed × 2.0   → 技能伤害翻倍
```

### 7.2 联动组合示例

| 组合名 | 角色A（施加状态） | 角色B（利用状态） | 效果 |
|--------|-----------------|-----------------|------|
| **毒刃连环** | 唐门暗器师（中毒） | 五毒手角色 | 中毒目标伤害 ×2 |
| **冰火双击** | 寒冰诀角色（冰冻） | 烈焰掌角色 | 冰冻目标碎冰爆伤 |
| **破甲斩杀** | 破甲锤装备（破甲） | 破影剑法角色 | 低防目标 + 斩杀线处决 |
| **标记猎杀** | 破影剑法（死印标记） | 其他高爆发角色 | 死印 -20% 减伤 + 标记额外伤害 |
| **冰域控场** | 寒冰诀（冰地形） | 五毒手（毒沼叠加） | 冰+毒双控，敌人无法移动且持续掉血 |
| **治疗净化** | 慈航普渡（净化友方） | 龟息功（低血触发护甲） | 组合防线 |

### 7.3 与旧联动系统的对比

| 维度 | 旧联动（已删除） | 新状态联动 |
|------|----------------|-----------|
| 触发条件 | 全图凑人头数 | 战场局部实际交互 |
| 策略空间 | 50人必然全凑满 | 需精心搭配功法/装备 |
| 性能影响 | 全图遍历 O(n²) | 仅在攻击/施法时检查目标 debuff，O(1) |
| 可玩性 | 数值加成无感 | 视觉上看到毒→爆、冰→碎，有反馈感 |
