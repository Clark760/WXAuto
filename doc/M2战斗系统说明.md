## 4. 核心实现说明

### 4.1 CombatManager（战斗调度）

**职责**
- 固定逻辑帧（默认 10fps）驱动全体战斗决策。
- 每逻辑帧执行：刷新存活缓存 -> 重建空间哈希 -> 更新分组焦点 -> 更新流场 -> 单位决策/执行 -> 结算。
- 任一方归零时立即结算，并停止后续单位逻辑与移动。

**关键信号**
- `battle_started`
- `battle_ended`
- `damage_resolved`
- `unit_died`

---

### 4.2 UnitCombat（战斗公式）

实现内容：
- 普攻与技能双轨（依据 MP 自动选择是否放技能）。
- 外功/内功伤害公式：
  - `伤害 = 攻击属性 × 技能倍率 × (100 / (100 + 防御)) × 随机(0.9~1.1)`
- 闪避公式：
  - `闪避率 = (防守方SPD - 攻击方SPD) / (防守方SPD + 100)`，并限制到 `0%~40%`
- 暴击：
  - 基础 5%，受悟性增益；暴伤基础 150%，受悟性微调。
- 内力：
  - 普攻命中回蓝，受击回蓝，技能消耗蓝量。

---

### 4.3 FlowField + UnitMovement（群体移动）

- `FlowField` 基于 Hex 网格做 BFS 反向扩散，输出“每格到目标群的下降梯度方向”。
- `CombatManager` 将方向下发给 `UnitMovement.set_flow_direction()`。
- `UnitMovement` 在渲染帧平滑推进目标点，实现“逻辑低频 + 渲染高频”。

---

### 4.4 VfxFactory（程序化特效）

- 从 `DataManager.get_all_records("vfx")` 读取特效参数。
- 通过 `ObjectPool` 复用：
  - `GPUParticles2D`（攻击剑气）
  - `Label`（伤害数字/闪避文本）
- 战斗事件触发：
  - 命中时播放 `vfx_sword_qi`
  - 同步弹出伤害数字；暴击放大、闪避显示“闪避”

## 5. 场景入口与操作

### 5.1 入口
- 主场景：`F6` -> `res://scenes/battle/battlefield_m2.tscn`
- 调试场景：`F6` -> `res://scenes/battle/battlefield_m2.tscn`

### 5.2 M2 快捷键
- `F3` 返回主场景
- `F4` 补充备战席角色
- `F6` 从准备阶段开始战斗
- `F7` 重开 M2 场景

---

## 6. 可复现验证步骤（M2）

1. 进入 M2 场景（`F6`）。
2. 准备阶段拖拽部分角色到左半场部署区。
3. 按 `F6` 开始战斗，观察：
   - 单位自动寻敌与移动。
   - 普攻/技能触发，伤害数字与粒子特效出现。
   - 角色死亡动画触发并回收。
4. 当任一方存活人数归零时，确认战斗立即结束，不再有单位继续位移或攻击。
5. 按 `F7` 重开，重复步骤 2~4 进行回归。

1. `UnitCombat` 可继续接入功法修正（倍率、暴击、减伤、吸血等）。

3. `VfxFactory` 已具备数据驱动入口，可在 M3 扩展更多 `vfx_id`。
