# M5 效果逻辑合并修正

> **目标读者**：AI 代码生成器。本文档分析如何将 Boss 战特殊机制合并进现有功法/装备/联动效果管线，避免重复造轮子。

---

## 一、现状分析

### 1.1 已有的效果执行管线

```
┌─────────────────────────────────────────────────────┐
│                GongfaManager                         │
│  prepare_battle → 为每个 unit 构建 state:            │
│    passive_effects[]  ← 来自功法 + 装备 + 联动       │
│    triggers[]         ← 来自功法技能 + 装备触发        │
│                                                      │
│  _process(delta):                                    │
│    BuffManager.tick() → 到期/周期效果                  │
│    _poll_auto_triggers() → 检查条件 → _try_fire_skill │
│    _refresh_linkages() → 联动检测 → 重算属性          │
├─────────────────────────────────────────────────────┤
│                GongfaEffectEngine                     │
│  被动 op:                                            │
│    stat_add / stat_percent / mp_regen_add            │
│    damage_reduce_flat / damage_reduce_percent        │
│    dodge_bonus / crit_bonus / attack_speed_bonus     │
│    range_add / 旧版属性别名                           │
│  主动 op:                                            │
│    damage_target / damage_aoe                        │
│    heal_self / heal_self_percent / heal_allies_aoe   │
│    buff_self / buff_allies_aoe                       │
│    debuff_target / debuff_aoe                        │
│    spawn_vfx                                         │
│  预留 op（已有分支，未实现）:                          │
│    teleport_behind / dash_forward / knockback_target │
│    summon_clone / revive_random_ally / taunt_aoe     │
├─────────────────────────────────────────────────────┤
│                BuffManager                           │
│  apply_buff(target, buff_id, duration, source)       │
│  tick(delta) → 到期移除 / 周期 tick_effects          │
│  buff 定义来自 data/buffs/*.json                     │
├─────────────────────────────────────────────────────┤
│                LinkageDetector                        │
│  detect_all(linkages, units, hex_grid) → results     │
│  联动效果 → 作为额外 passive_effects 叠加             │
└─────────────────────────────────────────────────────┘
```

### 1.2 M5 设计中的 Boss 机制（独立方案）

```
BossMechanicRunner → BaseMechanic 子类 ×6
  phase_shield_mechanic.gd
  summon_wave_mechanic.gd
  hazard_zones_mechanic.gd
  clone_split_mechanic.gd
  enrage_timer_mechanic.gd
  boss_formation_mechanic.gd
```

### 1.3 问题

**两套系统做了大量重复工作**：

| 能力 | GongfaManager 已有 | BossMechanicRunner 需重写 |
|------|:--:|:--:|
| HP 阈值触发 | ❌ 缺 | ✓ |
| 定时触发 | ❌ 缺 | ✓ |
| 造成伤害 | ✓ `damage_aoe` | 需重写 |
| 施加 Buff | ✓ `buff_self` | 需重写 |
| 属性修改 | ✓ `stat_add` 等 | 需重写 |
| 召唤单位 | ⚠ 已预留 `summon_clone` | 需重写 |
| VFX 播放 | ✓ `spawn_vfx` | 需重写 |

**结论**：Boss 机制应作为**特殊功法**挂载到 Boss 单位上，复用 GongfaEffectEngine。需要做的仅是扩展触发条件和补全几个预留 op。

## 二、合并方案

### 2.1 核心思路

> Boss 机制 = **特殊功法**（特殊触发条件 + 已有/新增 op 的组合）

每个 Boss 的特殊机制在关卡 JSON 中定义为一组**功法 ID**，在 `prepare_battle` 时自动注入到 Boss 单位的功法列表中。GongfaManager 统一处理。

### 2.2 需要扩展的 Trigger 类型

| 新 Trigger | 语义 | Boss 机制用途 |
|-----------|------|-------------|
| `on_hp_below` | 自身 HP 降至阈值时触发（一次性） | 相位护盾、分身幻术 |
| `on_time_elapsed` | 战斗经过 N 秒后触发（一次性） | 狂暴计时器 |
| `periodic_seconds` | 每 N 秒触发一次 | 小兵增援、地面灾害 |
| `on_buff_expired` | 指定 Buff 到期时触发 | 护盾碎裂后的虚弱 |

**实现位置**：`GongfaManager._poll_auto_triggers()` 中新增 match 分支。

### 2.3 需要补全/新增的 Effect Op

| Op | 语义 | 当前状态 | Boss 机制用途 |
|----|------|---------|-------------|
| `shield_self` | 给自身施加护盾（独立 HP 池） | **新增** | 相位护盾 |
| `immunity_self` | 自身免疫所有伤害（带时长） | **新增**（可用 buff 实现） | 相位护盾 |
| `summon_units` | 在指定区域生成单位 | **新增**（预留的 `summon_clone` 可扩展） | 小兵增援、分身幻术 |
| `hazard_zone` | 在战场创建持续伤害区域 | **新增** | 地面灾害 |
| `stat_percent` | 属性百分比加成 | ✓ 已有 | 狂暴计时器 |
| `buff_self` | 施加 Buff | ✓ 已有 | 所有机制 |
| `damage_aoe` | 区域伤害 | ✓ 已有 | 地面灾害 |

**实现位置**：`GongfaEffectEngine._execute_active_op()` 中新增 match 分支。

## 三、6 种 Boss 机制的功法化映射

### 3.1 「天罡护体」→ 功法: `boss_phase_shield`

```json
{
  "id": "boss_phase_shield",
  "name": "天罡护体",
  "type": "neigong",
  "passive_effects": [],
  "skills": [
    {
      "trigger": "on_hp_below",
      "trigger_params": { "threshold": 0.7 },
      "max_trigger_count": 1,
      "effects": [
        { "op": "shield_self", "value": 500 },
        { "op": "buff_self", "buff_id": "boss_immune", "duration": 99 },
        { "op": "spawn_vfx", "vfx_id": "vfx_shield", "at": "self" }
      ]
    },
    {
      "trigger": "on_buff_expired",
      "trigger_params": { "watch_buff_id": "boss_shield" },
      "max_trigger_count": 1,
      "effects": [
        { "op": "debuff_target", "buff_id": "boss_vulnerable", "duration": 5 },
        { "op": "buff_self", "buff_id": "boss_atk_up_20", "duration": -1 }
      ]
    }
  ]
}
```

**逻辑**：`on_hp_below(0.7)` 触发 → `shield_self(500)` 创建护盾 + `buff_self("boss_immune")` 免疫 → 护盾 HP 归零时 BuffManager 移除 `"boss_immune"` Buff → `on_buff_expired("boss_shield")` 触发虚弱 + 永久增攻。

### 3.2 「呼朋引伴」→ 功法: `boss_summon_wave`

```json
{
  "id": "boss_summon_wave",
  "name": "呼朋引伴",
  "skills": [
    {
      "trigger": "periodic_seconds",
      "trigger_params": { "interval": 15 },
      "max_trigger_count": 3,
      "effects": [
        { "op": "summon_units", "units": [
            { "unit_id": "minion_swordsman", "count": 6, "star": 1 },
            { "unit_id": "minion_archer", "count": 4, "star": 1 }
          ], "deploy": "around_self", "radius": 3
        },
        { "op": "spawn_vfx", "vfx_id": "vfx_summon", "at": "self" }
      ]
    }
  ]
}
```

**逻辑**：`periodic_seconds(15)` 每 15 秒触发 → `summon_units` 在 Boss 周围生成小兵。`max_trigger_count=3` 限制最多 3 波。

### 3.3 「天地为炉」→ 功法: `boss_hazard_zones`

```json
{
  "id": "boss_hazard_zones",
  "name": "天地为炉",
  "skills": [
    {
      "trigger": "periodic_seconds",
      "trigger_params": { "interval": 8 },
      "effects": [
        { "op": "hazard_zone",
          "hazard_type": "fire",
          "damage_per_second": 25,
          "radius_cells": 3,
          "duration": 7.5,
          "count": 2,
          "warning_seconds": 2.0,
          "target_mode": "random_position"
        }
      ]
    }
  ]
}
```

**逻辑**：`periodic_seconds(8)` 触发 → `hazard_zone` op 在随机位置创建火焰区域。`hazard_zone` 的存活管理委托给 `BuffManager`（创建一个特殊的"战场 Buff"，每周期 tick 对区域内单位调用 `damage_aoe`）。

### 3.4 「分身幻术」→ 功法: `boss_clone_split`

```json
{
  "id": "boss_clone_split",
  "name": "分身幻术",
  "skills": [
    {
      "trigger": "on_hp_below",
      "trigger_params": { "threshold": 0.5 },
      "max_trigger_count": 1,
      "effects": [
        { "op": "summon_units", "units": [
            { "clone_source": "self", "count": 2, "hp_ratio": 0.3, "atk_ratio": 0.5 }
          ], "deploy": "around_self", "radius": 2
        },
        { "op": "buff_self", "buff_id": "boss_stealth", "duration": 3 },
        { "op": "spawn_vfx", "vfx_id": "vfx_clone", "at": "self" }
      ]
    }
  ]
}
```

**逻辑**：`on_hp_below(0.5)` 触发 → `summon_units` 的 `clone_source="self"` 模式复制 Boss 外观与部分属性，生成 2 个分身。`buff_self("boss_stealth")` 短暂隐身。

### 3.5 「绝地反击」→ 功法: `boss_enrage`

```json
{
  "id": "boss_enrage",
  "name": "绝地反击",
  "skills": [
    {
      "trigger": "on_time_elapsed",
      "trigger_params": { "at_seconds": 30 },
      "max_trigger_count": 1,
      "effects": [
        { "op": "buff_self", "buff_id": "boss_soft_enrage_1", "duration": -1 }
      ]
    },
    {
      "trigger": "on_time_elapsed",
      "trigger_params": { "at_seconds": 45 },
      "max_trigger_count": 1,
      "effects": [
        { "op": "buff_self", "buff_id": "boss_soft_enrage_2", "duration": -1 }
      ]
    },
    {
      "trigger": "on_time_elapsed",
      "trigger_params": { "at_seconds": 60 },
      "max_trigger_count": 1,
      "effects": [
        { "op": "buff_self", "buff_id": "boss_full_enrage", "duration": -1 },
        { "op": "spawn_vfx", "vfx_id": "vfx_enrage", "at": "self" }
      ]
    }
  ]
}
```

**逻辑**：三个 `on_time_elapsed` 触发器分别在 30s/45s/60s 施加递增 Buff。Buff 定义在 `data/buffs/` 中包含属性加成（`atk +30%/+60%/+100%` 等）。完全复用现有 Buff 系统。

### 3.6 「天罡阵法」→ 联动: `boss_formation_linkage`

```json
{
  "id": "linkage_boss_formation",
  "name": "天罡阵法",
  "type": "formation_boost",
  "conditions": {
    "require_zhenfa_tag": "boss_formation_master",
    "require_any_tag": ["boss_formation_master"],
    "min_count": 1
  },
  "effects": [
    { "op": "damage_reduce_percent", "value": 0.5, "target": "participants" },
    { "op": "hp_regen_add", "value": 20, "target": "participants" }
  ]
}
```

**逻辑**：Boss 自带 `runtime_linkage_tags: ["boss_formation_master"]`。`LinkageDetector` 自动检测到满足条件 → 联动效果为减伤 50% + 回血。**完全复用现有联动系统，零新代码**。

若要实现"被推出阵法区域则失效"的效果，可将 `require_adjacent: true` 结合特定标记格实现。

## 四、需修改的文件清单

### 4.1 `GongfaManager` — 新增 trigger 类型

| 修改点 | 说明 |
|--------|------|
| `_poll_auto_triggers()` | 新增 `on_hp_below`、`on_time_elapsed`、`periodic_seconds` 的 match 分支 |
| `_on_damage_resolved()` 或 `_process()` | `on_hp_below` 需监听单位 HP 变化并与阈值比较 |
| `_process()` | `periodic_seconds` 需按 `_battle_elapsed` 计算下次触发时间 |
| `_on_buff_removed()` | `on_buff_expired` 需监听 BuffManager 的 Buff 移除事件 |

### 4.2 `GongfaEffectEngine` — 新增 op

| 修改点 | 说明 |
|--------|------|
| `_execute_active_op()` | 新增 `shield_self`、`summon_units`、`hazard_zone` 的 match 分支 |
| `shield_self` | 创建护盾需与 `UnitCombat` 协作（新增 `shield_hp` 属性） |
| `summon_units` | 需调用 `UnitFactory.acquire_unit()` 并注册到 `CombatManager` |
| `hazard_zone` | 需创建战场级持续区域，可委托 `BuffManager` 管理生命周期 |

### 4.3 `BuffManager` — 新增能力

| 修改点 | 说明 |
|--------|------|
| 新增信号 `buff_removed` | 供 `on_buff_expired` trigger 监听 |
| 支持"战场级 Buff" | `hazard_zone` 不绑定到 unit 而绑定到 position+radius |

### 4.4 `UnitCombat` — 护盾支持

| 修改点 | 说明 |
|--------|------|
| 新增 `shield_hp` / `max_shield_hp` 属性 | 受伤时优先扣护盾 |
| `receive_damage()` | 先扣护盾再扣 HP |

### 4.5 关卡 JSON 变化

原来的 `boss_mechanics` 字段**改为 `boss_gongfa_ids`**：

```json
{
  "boss_mechanics": null,
  "boss_gongfa_ids": ["boss_phase_shield", "boss_summon_wave"]
}
```

`StageManager` 在关卡加载时将这些功法 ID 注入到 Boss 单位的 `equipped_gongfa_ids` 中，其余完全由 GongfaManager 接管。

## 六、架构对比

### 合并前

```
GongfaManager ─── 功法被动/技能/Buff/联动
BossMechanicRunner ─── Boss 独立机制 ×6
（两套独立的效果执行管线，大量重复逻辑）
```

### 合并后

```
GongfaManager ─── 功法被动/技能/Buff/联动/Boss 机制
                  ├── 被动 op：stat_add, shield_self, ...
                  ├── 主动 op：damage_aoe, summon_units, hazard_zone, ...
                  ├── 触发器：on_attack, on_hp_below, periodic_seconds, ...
                  └── 联动：faction_combo, boss_formation, ...
（统一管线，Boss 机制 = 特殊功法 JSON）
```

### 优势

| 项目 | 合并前 | 合并后 |
|------|--------|--------|
| 新增脚本文件数 | 13 | 0 |
| Boss 机制可 Mod 化 | ❌ 需改代码 | ✓ 只改 JSON |
| 与 Buff/联动交互 | 需手动桥接 | 自动生效 |
| 测试复杂度 | 需测两套管线 | 只测一套 |
| Boss 可使用现有功法 | ❌ | ✓ Boss 可装备任意功法 |

## 七、实施顺序

| 步骤 | 改动 | 依赖 |
|------|------|------|
| 1 | `GongfaManager` 新增 `on_hp_below` / `on_time_elapsed` / `periodic_seconds` trigger | 无 |
| 2 | `BuffManager` 新增 `buff_removed` 信号，支持 `on_buff_expired` | 步骤 1 |
| 3 | `GongfaEffectEngine` 补全 `summon_units` op | `UnitFactory` 接口 |
| 4 | `UnitCombat` 新增 `shield_hp`，`EffectEngine` 新增 `shield_self` | 无 |
| 5 | `GongfaEffectEngine` 新增 `hazard_zone` op | `BuffManager` 战场 Buff |
| 6 | 编写 Boss 功法 JSON + Boss 联动 JSON | 步骤 1-5 |
| 7 | `StageManager` 在加载关卡时注入 `boss_gongfa_ids` | 步骤 6 |
