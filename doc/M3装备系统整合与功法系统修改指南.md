# M3 装备与功法系统修改指南（面向代码 AI）

> **背景**：装备从 M5 并入 M3。装备与功法本质相同：都是"附着在角色身上、提供被动效果、可能带触发器"的数据实体，完全复用 `EffectEngine`、`BuffManager`、`LinkageDetector`。
> **M3 新增需求**：实现装备与功法仓库 UI（背包面板），用于管理玩家拥有的所有装备和功法。
> **Mod 文档**：配套 `doc/Mod创作指南_装备功法联动.md` 供 Mod 作者参考。

## 一、代码审查结论

### 1.1 现有系统能力矩阵

| 系统 | 能力 | 装备可复用 |
|------|------|-----------|
| `GongfaRegistry` | 从 DataManager 加载 gongfa/linkages/buffs 三类数据 | ✅ 加 equipment 类 |
| `EffectEngine` | 10种被动op + 14种主动op | ✅ 完全复用 |
| `BuffManager` | 叠加/刷新/过期/tick_effects | ✅ 完全复用 |
| `LinkageDetector` | 4种联动类型检测 | ✅ 不受影响 |
| `GongfaManager` | 被动效果→属性重算→技能触发→联动→Buff | ⚠️ 需扩展装备层 |

### 1.2 现有系统缺失项

| 缺失项 | 说明 |
|--------|------|
| **装备数据格式** | `sample_equipment.json` 的 `passive` 是纯文本字符串，无法被 `EffectEngine` 执行 |
| **装备属性叠加层** | `_apply_state_to_unit` 只有 `passive_effects` + `linkage_effects` + `buff_effects` 三层，缺少 `equipment_effects` |
| **装备槽位字段** | `unit_data.gd` 无 `equip_slots`（weapon/armor/accessory） |
| **装备穿脱接口** | `GongfaManager` 只有 `equip_gongfa` / `unequip_gongfa`，缺少装备等价接口 |
| **装备数据加载** | `GongfaRegistry` 只加载 gongfa/linkages/buffs 三类，缺少 equipment |

## 二、装备数据格式升级

### 2.1 当前格式（不可执行）

```json
{
  "id": "equip_qinggang_jian",
  "name": "青钢剑",
  "type": "weapon",
  "rarity": "green",
  "stats": { "atk": 22, "spd": 6 },
  "recipe": [],
  "passive": "普通攻击有 10% 概率附加 1 层流血"  ← 纯文本，无法执行
}
```

### 2.2 目标格式（与功法统一）

```json
{
  "id": "equip_qinggang_jian",
  "name": "青钢剑",
  "description": "青锋出鞘，剑意如虹。",
  "type": "weapon",
  "element": "metal",
  "rarity": "green",
  "passive_effects": [
    { "op": "stat_add", "stat": "atk", "value": 22 },
    { "op": "stat_add", "stat": "spd", "value": 6 }
  ],
  "trigger": {
    "type": "on_attack_hit",
    "chance": 0.10,
    "cooldown": 3.0,
    "effects": [
      { "op": "debuff_target", "buff_id": "buff_bleed", "duration": 5.0 }
    ]
  },
  "recipe": [],
  "linkage_tags": ["青钢", "金系兵器"]
}
```

### 2.3 关键对齐规则

| 字段 | 功法 | 装备 | 说明 |
|------|------|------|------|
| `id` | `gongfa_xxx` | `equip_xxx` | 前缀区分 |
| `type` | `neigong/waigong/...` | `weapon/armor/accessory` | 决定槽位 |
| `passive_effects` | ✅ 数组 | ✅ 改为数组（取代旧 `stats` 字典） | `EffectEngine.apply_passive_effects()` 直接执行 |
| `skill` / `trigger` | 功法用 `skill` | 装备用 `trigger`（结构相同） | 统一复用 `_try_fire_skill` |
| `linkage_tags` | ✅ | ✅ 新增 | 装备也可参与联动 |
| `description` | ✅ | ✅ 新增 | UI 展示用 |

### 2.4 旧格式兼容

为兼容已有的 `stats` 格式，在加载时自动转换为 `passive_effects`：

```gdscript
# 在 GongfaRegistry 或加载阶段执行
static func _migrate_equipment_stats(record: Dictionary) -> Dictionary:
    if record.has("passive_effects"):
        return record  # 新格式，无需转换
    var stats: Variant = record.get("stats", {})
    if stats is Dictionary:
        var effects: Array[Dictionary] = []
        for key in (stats as Dictionary).keys():
            effects.append({
                "op": "stat_add",
                "stat": str(key),
                "value": float((stats as Dictionary)[key])
            })
        record["passive_effects"] = effects
    # 旧 passive 文本留作 description
    if record.has("passive") and not record.has("description"):
        record["description"] = record["passive"]
    return record
```

#### 3.3.4 新增装备穿脱接口

```gdscript
func equip_equipment(unit: Node, slot: String, equip_id: String) -> bool:
    if unit == null or not is_instance_valid(unit):
        return false
    if not ALLOWED_EQUIP_SLOTS.has(slot):
        return false
    if equip_id.strip_edges().is_empty():
        return false
    if not _registry.has_equipment(equip_id):
        return false
    
    var equip_data: Dictionary = _registry.get_equipment(equip_id)
    var equip_type: String = str(equip_data.get("type", ""))
    # 校验类型与槽位匹配
    if equip_type != slot:
        return false
    
    var slots: Dictionary = _normalize_equip_slots_dict(_node_prop(unit, "equip_slots", {}))
    slots[slot] = equip_id
    unit.set("equip_slots", slots)
    apply_gongfa(unit)  # 全部重算
    _refresh_linkages(true)
    return true


func unequip_equipment(unit: Node, slot: String) -> void:
    if unit == null or not is_instance_valid(unit):
        return
    if not ALLOWED_EQUIP_SLOTS.has(slot):
        return
    var slots: Dictionary = _normalize_equip_slots_dict(_node_prop(unit, "equip_slots", {}))
    slots[slot] = ""
    unit.set("equip_slots", slots)
    apply_gongfa(unit)
    _refresh_linkages(true)


func get_equipment_data(equip_id: String) -> Dictionary:
    return _registry.get_equipment(equip_id)


func get_all_equipment() -> Array[Dictionary]:
    return _registry.get_all_equipment()


func _resolve_equipped_equip_ids(unit: Node) -> Array[String]:
    var ids: Array[String] = []
    var slots: Dictionary = _normalize_equip_slots_dict(_node_prop(unit, "equip_slots", {}))
    for slot in ALLOWED_EQUIP_SLOTS:
        var eid: String = str(slots.get(slot, "")).strip_edges()
        if eid.is_empty():
            continue
        if ids.has(eid):
            continue
        ids.append(eid)
    return ids


func _normalize_equip_slots_dict(raw: Variant) -> Dictionary:
    var slots: Dictionary = {"weapon": "", "armor": "", "accessory": ""}
    if raw is Dictionary:
        for slot in ALLOWED_EQUIP_SLOTS:
            slots[slot] = str((raw as Dictionary).get(slot, "")).strip_edges()
    return slots
```

```gdscript
func _on_equip_swap_pressed(equip_type: String) -> void:
    if _detail_unit == null or not _is_valid_unit(_detail_unit):
        return
    if _stage != Stage.PREPARATION:
        return
    if gongfa_manager == null:
        return
    
    var equip_slots: Dictionary = _normalize_equip_slots(_get_unit_equip_slots(_detail_unit))

    var current_index: int = candidates.find(current_id)

    else:

    _refresh_all_ui()
```

## 四、属性叠加层顺序

修改后 `_apply_state_to_unit` 的四层叠加顺序：

```
baseline_stats（角色基础 × 星级）
  ↓ 第1层 apply_passive_effects  ← 功法被动（passive_effects）
  ↓ 第2层 apply_passive_effects  ← 装备被动（equipment_effects）
  ↓ 第3层 apply_passive_effects  ← 联动效果（linkage_effects）
  ↓ 第4层 apply_passive_effects  ← Buff效果（buff_manager.collect）
  ↓ clamp_runtime_stats
  ↓ 写入 unit.runtime_stats
```

> `stat_add` 按顺序累加，`stat_percent` 按顺序乘算。这意味着：
> - 功法的百分比增加基于基础值
> - 装备的百分比增加基于（基础+功法）
> - 联动的百分比增加基于（基础+功法+装备）

## 五、代码复用架构

装备、功法、联动触发的 Buff 和效果共享同一套执行引擎：

```
┌───────────────────────────────────────────────────────────────┐
│                  统一效果执行管线                               │
│                                                               │
│   数据来源          效果数组          执行引擎                   │
│  ┌──────────┐    ┌───────────┐    ┌──────────────┐           │
│  │ 功法 JSON │─→ │ passive_  │─→ │              │           │
│  │ gongfa/* │    │ effects   │    │ EffectEngine │           │
│  ├──────────┤    ├───────────┤    │              │           │
│  │ 装备 JSON │─→ │ passive_  │─→ │ apply_passive│           │
│  │equipment/│    │ effects   │    │ _effects()   │           │
│  ├──────────┤    ├───────────┤    │              │           │
│  │ 联动 JSON │─→ │ effects   │─→ │              │           │
│  │linkages/ │    │ (联动效果) │    │ execute_     │           │
│  ├──────────┤    ├───────────┤    │ active_      │           │
```

**关键复用点**：

| 组件 | 被谁调用 | 说明 |
|------|---------|------|
| `EffectEngine.apply_passive_effects()` | 功法被动、装备被动、联动效果、Buff 效果 | 四个来源的 effects 数组格式完全相同 |
| `EffectEngine.execute_active_effects()` | 功法技能、装备触发器、Buff tick_effects | 三个来源共享主动效果执行 |
| `BuffManager.apply_buff()` | 功法技能中的 `buff_self`、装备触发器中的 `debuff_target`、联动效果 | buff_id 统一引用 `data/buffs/*.json` |
| `_try_fire_skill()` in GongfaManager | 功法 triggers 和装备 triggers 合并到同一数组 | 共享触发检测、冷却、概率判定 |

> **新增数据类型时不需要修改 EffectEngine / BuffManager / LinkageDetector**。
> 只需：1) 在 JSON 中定义数据；2) 在 GongfaManager 中收集 effects → 传入 EffectEngine。

### 6.4 代码 AI 实现指南

```gdscript
# 在 battlefield_m3.gd 中新增：

var _inventory_mode: String = ""  # "gongfa" 或 "equipment"

func _open_inventory(mode: String) -> void:
    _inventory_mode = mode
    inventory_panel.visible = true
    _rebuild_inventory_items()

func _rebuild_inventory_items() -> void:
    # 清空旧卡片
    for child in inventory_grid.get_children():
        child.queue_free()
    
    var items: Array = []
    if _inventory_mode == "gongfa":
        items = gongfa_manager.call("get_all_gongfa")
    else:
        items = gongfa_manager.call("get_all_equipment")
    
    # 按品质排序
    items.sort_custom(_sort_by_quality)
    
    # 应用筛选
    var filter_type: String = _current_filter_type
    if not filter_type.is_empty() and filter_type != "all":
        items = items.filter(func(item): return str(item.get("type", "")) == filter_type)
    
    var search_text: String = inventory_search.text.strip_edges().to_lower()
    if not search_text.is_empty():
        items = items.filter(func(item): return str(item.get("name", "")).to_lower().contains(search_text))
    
    for item_data in items:
        var card := _create_item_card(item_data)
        inventory_grid.add_child(card)
    
    _update_inventory_summary(items.size())

func _create_item_card(item_data: Dictionary) -> PanelContainer:
    var card := PanelContainer.new()
    card.custom_minimum_size = Vector2(140, 100)
    # ... 创建子节点 IconLabel / NameLabel / TypeLabel / StatusLabel ...
    # 查找谁装备了此物品
    var equipped_by: String = _find_equipped_by(str(item_data.get("id", "")))
    # StatusLabel 显示 "已装备:{角色名}" 或 "未装备"
    # 连接点击信号
    card.gui_input.connect(_on_inventory_card_input.bind(card, item_data))
    # 连接悬浮信号（复用 ItemTooltip）
    card.mouse_entered.connect(_on_item_source_hover_entered.bind(card, _build_tooltip_data(item_data)))
    card.mouse_exited.connect(_on_item_source_hover_exited.bind(card))
    return card
```
