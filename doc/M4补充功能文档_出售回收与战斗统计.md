## 一、出售回收区

### 1.1 功能描述

玩家可将**备战区角色**、**仓库中的功法**、**仓库中的装备**拖入一个固定的"回收区"来出售，获得银两（Silver）。

### 1.2 回收区 UI

回收区位于底部备战区的右侧，作为一个**常驻回收槽**（BottomLayer 内）。

```text
┌─── 备战区 ──────────────────────────────────────────┬──────────┐
│ [角色1] [角色2] [角色3] ... [角色N]                    │  🗑️ 回收  │
│                                                      │  出售获得  │
│                                                      │  💰 ?     │
└──────────────────────────────────────────────────────┴──────────┘
```

### 1.3 场景树位置

```
BottomLayer (CanvasLayer, layer=5)
└── BottomPanel
    ├── BenchArea (ScrollContainer)
    ├── RecycleDropZone (PanelContainer)   ← ★ 新增
    │   ├── RecycleIcon (Label "🗑️")
    │   ├── RecycleHint (Label "拖入出售")
    │   └── RecyclePrice (Label "💰 ?")   ← 动态显示预估售价
    ├── ShopBar
    └── ResourceBar
```

### 1.4 拖放交互

| 拖放源 | 拖放目标 | 行为 |
|--------|----------|------|
| 备战区角色 | RecycleDropZone | 出售角色，获得银两，角色从备战区移除 |
| 仓库功法卡片 | RecycleDropZone | 出售功法，获得银两，功法从仓库移除 |
| 仓库装备卡片 | RecycleDropZone | 出售装备，获得银两，装备从仓库移除 |
| 已上场角色 | RecycleDropZone | **禁止**，显示 🚫（必须先下场到备战区再卖） |

### 1.5 售价规则

```gdscript
# 售价 = 角色原始 cost（与购买价相同，自走棋标准设定）
func get_sell_price_unit(unit: Node) -> int:
    return int(unit.get("cost"))

# 功法/装备售价 = 品质对应固定值
const QUALITY_SELL_PRICE: Dictionary = {
    "white": 1, "green": 2, "blue": 3,
    "purple": 5, "orange": 8, "red": 15
}

func get_sell_price_item(item_data: Dictionary) -> int:
    var q: String = str(item_data.get("quality", item_data.get("rarity", "white")))
    return QUALITY_SELL_PRICE.get(q, 1)
```

### 1.6 实现要点（代码 AI 指导）

#### 1.6.1 RecycleDropZone 脚本

新建 `scripts/ui/m3_recycle_drop_zone.gd`，实现 `_can_drop_data` 和 `_drop_data`：

```gdscript
extends PanelContainer

signal item_sold(item_type: String, item_id: String, price: int)

var _price_label: Label = null

func _can_drop_data(_pos: Vector2, data: Variant) -> bool:
    if not (data is Dictionary):
        return false
    var d: Dictionary = data as Dictionary
    # 接受 unit / gongfa / equipment
    var type: String = str(d.get("type", ""))
    if type in ["unit", "gongfa", "equipment"]:
        # 动态显示预估售价
        _update_price_preview(d)
        modulate = Color(1.0, 0.85, 0.85, 1.0)  # 红色高亮
        return true
    return false

func _drop_data(_pos: Vector2, data: Variant) -> void:
    if not (data is Dictionary):
        return
    var d: Dictionary = data as Dictionary
    var type: String = str(d.get("type", ""))
    var id: String = str(d.get("id", ""))
    var price: int = _calc_price(d)
    item_sold.emit(type, id, price)
    _reset_visual()

func _notification(what: int) -> void:
    if what == NOTIFICATION_DRAG_END:
        _reset_visual()
```

#### 1.6.2 battlefield_m3.gd 修改

1. 在 `_ensure_bottom_panel_created` 中创建 `RecycleDropZone` 并挂到 `BottomPanel` 右侧。
2. 连接 `item_sold` 信号 → `_on_item_sold(type, id, price)`。
3. `_on_item_sold` 中：
   - `type == "unit"` → 从备战区移除角色节点、回收到对象池、add_silver(price)。
   - `type == "gongfa"` → 从仓库中移除（或标记为已售）、add_silver(price)。
   - `type == "equipment"` → 同上。
4. 刷新仓库 UI 和银两显示。

#### 1.6.3 拖放数据格式补充

备战区角色的 `_get_drag_data` 需要返回 `type: "unit"`：

```gdscript
# 备战区角色拖放源
return {
    "type": "unit",
    "id": unit.unit_id,
    "unit_node": unit,
    "cost": unit.cost
}
```

## 二、战斗统计系统

### 2.1 功能描述

战斗中记录**每个角色**的以下关键数据：
- **造成伤害** (damage_dealt)
- **承受伤害** (damage_taken)
- **治疗量** (healing_done)

战斗结束后可在结算面板中展示 MVP、DPS 排行等。

### 2.2 数据结构

```gdscript
# BattleStatisticsManager 或直接挂在 CombatManager 上
var _unit_stats: Dictionary = {}  # unit_instance_id -> stats

func _init_unit_stats(unit: Node) -> void:
    _unit_stats[unit.get_instance_id()] = {
        "unit_name": unit.unit_name,
        "unit_id": unit.unit_id,
        "team_id": unit.team_id,
        "damage_dealt": 0,
        "damage_taken": 0,
        "healing_done": 0,
        "kills": 0,
        "deaths": 0
    }
```

### 2.3 数据采集点

需要在以下位置插入统计钩子：

| 事件 | 采集文件 | 采集方式 |
|------|----------|----------|
| 普攻造成伤害 | `unit_combat.gd` | 在 `_apply_attack_damage` 后记录 |
| 技能造成伤害 | `gongfa_manager.gd` → `_execute_skill_effects` | 在 `damage_target`/`damage_aoe` 效果执行后记录 |
| Buff tick 伤害 | `buff_manager.gd` → tick | 在 `damage_target` tick 执行后记录 |
| 治疗 | `gongfa_manager.gd` → `_execute_skill_effects` | 在 `heal_self`/`heal_self_percent`/`heal_allies_aoe` 执行后记录 |
| Buff tick 治疗 | `buff_manager.gd` → tick | 在 `heal_self` tick 执行后记录 |
| 角色死亡 | `unit_combat.gd` | 在 HP 归零后记录 kills/deaths |

## 三、与现有系统的关系

- **出售回收区**依赖 `EconomyManager.add_silver()` 接口（M4 经济系统）。
- **战斗统计**独立于经济系统，但结算面板可在同一 UI 层级（DetailLayer）展示。
- 两个功能均在**布阵期**和**交锋结束后**分别激活，互不冲突。
