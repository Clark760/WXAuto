# 战斗日志规划

> 目标：在 50v200 高密度战斗下，实时显示「谁对谁造成了多少伤害」和「谁发动了什么主动技能」，同时将 FPS 影响控制在可接受范围内。

---

## 1 当前架构分析

### 1.1 现有日志链路

```
CombatManager signals (damage_resolved / unit_died / …)
  ↓
BattlefieldCoordinator._on_damage_resolved()  → 只写 statistics，不写日志
  ↓
_append_battle_log()  → 目前只在关卡/系统事件时调用
  ↓
BattlefieldHudPresenter.append_battle_log()
  ↓
BattlefieldHudRuntimeView.append_battle_log()
  → 写入 state.battle_log_entries (Array[String])
  → 按 0.12s 间隔 flush 到 RichTextLabel
```

### 1.2 当前瓶颈点

| 层级 | 问题 | 影响 |
|------|------|------|
| **事件产生** | 50v200 场景下每逻辑帧（10 fps）可能产生 50~200 条伤害事件 | 信号回调密集 |
| **字符串拼接** | 每条事件都做 BBCode 颜色包装 + 字符串 join | GC 压力、分配开销 |
| **RichTextLabel** | `append_text` 每次调用都会触发 BBCode 解析和布局重排 | UI 树脏标记频繁 |
| **日志滚动** | `scroll_to_line` 在行数多时开销线性增长 | 滚动卡顿 |
| **缓存上限** | `BATTLE_LOG_MAX_LINES = 50`，裁剪时 `remove_at(0)` 触发数组搬移 + 全量 rebuild | 阵列式开销 |

### 1.3 估算：50v200 的日志吞吐量

- 逻辑帧率 10 fps，每帧约 50 个单位可发起攻击
- 每秒约 50~100 条伤害事件 + 5~15 条技能事件
- 如果逐条写入 RichTextLabel → 每秒 ~115 次 UI 树变更 → **严重帧率下降**

---

## 2 设计方案：三级日志架构

将日志拆成**采集层 → 聚合层 → 渲染层**三级管线，每一级都可独立节流。

```
┌──────────┐     ┌───────────┐     ┌──────────────┐
│  采集层  │────→│  聚合层   │────→│  渲染层      │
│ (信号回调)│     │ (环形缓冲)│     │ (定时 flush) │
└──────────┘     └───────────┘     └──────────────┘
                       ↓
              ┌────────────────┐
              │ 合并 & 摘要策略 │
              └────────────────┘
```

### 2.1 采集层：事件分类 + 预过滤

在 `BattlefieldCoordinator` 的 `_on_damage_resolved` / `_on_skill_effect_damage` 等回调中，将战斗事件追加到一个**结构化事件队列**，而非直接拼 BBCode 字符串。

**事件结构体：**

```gdscript
# 轻量事件记录，不做任何 UI 操作
var _combat_log_ring: Array[Dictionary] = []

# 事件类型参考
enum LogEventType {
    DAMAGE,         # 普通伤害
    SKILL_DAMAGE,   # 技能伤害
    SKILL_HEAL,     # 技能治疗
    SKILL_CAST,     # 技能释放（无伤害）
    DEATH,          # 单位阵亡
    BUFF_APPLIED,   # Buff 施加
    SYSTEM,         # 系统消息
}

# 单条事件格式
{
    "type": LogEventType.DAMAGE,
    "source_name": "张三丰",
    "source_team": 1,
    "target_name": "黑风寨匪首",
    "target_team": 2,
    "value": 234,              # 伤害/治疗值
    "skill_name": "",          # 技能名（如有）
    "timestamp": 12.45,        # 逻辑战斗时钟
}
```

**预过滤规则：**

| 条件 | 策略 |
|------|------|
| 伤害值 < 阈值（如 10） | 不入队列，只计入统计 |
| 同源同目标连续伤害 | 标记为可合并 |
| 环形缓冲溢出 | 覆盖最旧条目，O(1) |

### 2.2 聚合层：环形缓冲 + 合并策略

用 **环形缓冲区**（Ring Buffer）替代现有 `Array[String]`：

```gdscript
# battle_log_ring_buffer.gd
extends RefCounted

const RING_CAPACITY: int = 128  # 2 的幂，方便取模

var _buffer: Array[Dictionary] = []
var _write_head: int = 0
var _read_head: int = 0
var _count: int = 0


func push(event: Dictionary) -> void:
    if _buffer.size() < RING_CAPACITY:
        _buffer.append(event)
    else:
        _buffer[_write_head] = event
    _write_head = (_write_head + 1) % RING_CAPACITY
    _count = mini(_count + 1, RING_CAPACITY)


func drain_since(last_read: int) -> Array[Dictionary]:
    var result: Array[Dictionary] = []
    var start: int = maxi(last_read, _write_head - _count)
    if start < 0:
        start += RING_CAPACITY
    while start != _write_head:
        result.append(_buffer[start % RING_CAPACITY])
        start = (start + 1) % RING_CAPACITY
    return result


func get_write_head() -> int:
    return _write_head
```

**合并规则（聚合窗口 = 0.3 ~ 0.5 秒）：**

| 类型 | 合并策略 | 展示示例 |
|------|----------|----------|
| 同源→同目标连续伤害 | 累加伤害值，记录次数 | `张三丰 → 匪首 共 3 击 567 伤` |
| 同源→多目标（AOE） | 按源聚合，列出目标 | `张三丰 [太极剑法] → 3 人 共 1204 伤` |
| 多源→同目标（集火） | 不合并但降频 | 只显示 top3 伤害来源 |
| 相同事件短期重复 | 增加计数后缀 | `张三丰 → 匪首 234 伤 ×3` |

### 2.3 渲染层：分帧写入 + 视口外跳过

**核心策略：**

| 优化点 | 做法 | 预期收益 |
|--------|------|----------|
| **降低 flush 频率** | 战斗期从 0.12s 提升到 **0.25~0.4s**，高密度（>120 单位）提升到 **0.5s** | UI 树变更次数降 50~75% |
| **批量写入** | 每次 flush 最多写入 **8~12 条**聚合后的文案 | 单次 append_text 调用减少 |
| **裁剪用换页** | 达到显示上限时不 remove_at(0)，而是整体 clear + 重写最近 N 条 | 避免逐条移除的 O(n) |
| **面板隐藏跳过** | `battle_log_panel.visible == false` 时完全跳过 flush | 面板关闭时零 UI 开销 |
| **延迟滚动** | scroll_to_line 只在用户未手动滚动时才执行 | 减少强制布局 |
| **预分配 BBCode** | 颜色 tag 用预构建的常量拼接，不每次 format | 减少 String 分配 |

**渲染层代码结构（融入现有 runtime_view）：**

```gdscript
# 在 battlefield_hud_runtime_view.gd 中修改

const BATTLE_LOG_FLUSH_INTERVAL_NORMAL: float = 0.25
const BATTLE_LOG_FLUSH_INTERVAL_DENSE: float = 0.50
const BATTLE_LOG_MAX_FLUSH_BATCH: int = 10
const BATTLE_LOG_DISPLAY_LINES: int = 40      # 面板可见行数上限
const DENSE_LOG_UNIT_THRESHOLD: int = 120

var _user_scrolled: bool = false  # 标记用户是否手动翻看日志


func update_battle_log_view(delta: float) -> void:
    if not _state.battle_log_dirty:
        return
    # 面板不可见时不做任何 UI 工作
    if _refs.battle_log_panel == null or not _refs.battle_log_panel.visible:
        return
    _state.battle_log_flush_accum += delta
    var interval: float = _resolve_log_flush_interval()
    if _state.battle_log_flush_accum < interval:
        return
    flush_battle_log(not _user_scrolled)


func _resolve_log_flush_interval() -> float:
    var total_units: int = 0
    if _state != null:
        total_units = _state.ally_deployed.size() + _state.enemy_deployed.size()
    if total_units >= DENSE_LOG_UNIT_THRESHOLD:
        return BATTLE_LOG_FLUSH_INTERVAL_DENSE
    return BATTLE_LOG_FLUSH_INTERVAL_NORMAL
```

## 4 核心代码建议

### 4.1 战斗日志聚合器

```gdscript
# scripts/app/battlefield/battlefield_combat_log_aggregator.gd
extends RefCounted

## 战斗日志聚合器
## 对高频战斗事件做窗口聚合后再输出可读文案，减少 UI 层写入压力。

const MERGE_WINDOW_SECONDS: float = 0.35   # 合并窗口时长
const MIN_DAMAGE_THRESHOLD: int = 5        # 低于此值的伤害不单独显示
const MAX_PENDING_EVENTS: int = 64         # 待聚合事件上限

# 事件类型常量
const TYPE_DAMAGE: int = 0
const TYPE_SKILL_DAMAGE: int = 1
const TYPE_SKILL_HEAL: int = 2
const TYPE_SKILL_CAST: int = 3
const TYPE_DEATH: int = 4
const TYPE_SYSTEM: int = 5

var _pending_events: Array[Dictionary] = []
var _window_start_time: float = 0.0
var _text_support = null  # battlefield_hud_text_support 引用

func initialize(text_support) -> void:
    _text_support = text_support


func push_event(event: Dictionary) -> void:
    # 低伤害过滤
    var event_type: int = int(event.get("type", TYPE_SYSTEM))
    if event_type == TYPE_DAMAGE or event_type == TYPE_SKILL_DAMAGE:
        if int(event.get("value", 0)) < MIN_DAMAGE_THRESHOLD:
            return

    if _pending_events.size() >= MAX_PENDING_EVENTS:
        _pending_events.remove_at(0)
    _pending_events.append(event)


## 按合并窗口输出聚合后的文案行。
## 返回 Array[Dictionary]，每个元素包含 { "text": String, "event_type": String }
func flush_aggregated(current_time: float) -> Array[Dictionary]:
    if _pending_events.is_empty():
        return []

    var output: Array[Dictionary] = []
    var merged: Dictionary = {}  # "source_name|target_name|type" -> merged event

    for event in _pending_events:
        var event_type: int = int(event.get("type", TYPE_SYSTEM))
        
        match event_type:
            TYPE_DAMAGE, TYPE_SKILL_DAMAGE:
                var key: String = "%s|%s|%d" % [
                    str(event.get("source_name", "")),
                    str(event.get("target_name", "")),
                    event_type
                ]
                if merged.has(key):
                    merged[key]["value"] = int(merged[key]["value"]) + int(event.get("value", 0))
                    merged[key]["count"] = int(merged[key]["count"]) + 1
                    # 保留技能名（取最后一次）
                    var skill: String = str(event.get("skill_name", ""))
                    if not skill.is_empty():
                        merged[key]["skill_name"] = skill
                else:
                    merged[key] = event.duplicate()
                    merged[key]["count"] = 1
            
            TYPE_DEATH:
                # 死亡事件不合并，直接输出
                output.append({
                    "text": _format_death_event(event),
                    "event_type": "death"
                })
            
            TYPE_SKILL_CAST:
                output.append({
                    "text": _format_skill_cast_event(event),
                    "event_type": "skill"
                })
            
            TYPE_SKILL_HEAL:
                var key: String = "heal|%s|%s" % [
                    str(event.get("source_name", "")),
                    str(event.get("target_name", ""))
                ]
                if merged.has(key):
                    merged[key]["value"] = int(merged[key]["value"]) + int(event.get("value", 0))
                    merged[key]["count"] = int(merged[key]["count"]) + 1
                else:
                    merged[key] = event.duplicate()
                    merged[key]["count"] = 1
            
            TYPE_SYSTEM:
                output.append({
                    "text": str(event.get("text", "")),
                    "event_type": "system"
                })

    # 把合并后的伤害/治疗事件转成文案
    for key in merged.keys():
        var m: Dictionary = merged[key]
        var event_type: int = int(m.get("type", TYPE_DAMAGE))
        
        if event_type == TYPE_DAMAGE or event_type == TYPE_SKILL_DAMAGE:
            output.append({
                "text": _format_damage_event(m),
                "event_type": "damage" if event_type == TYPE_DAMAGE else "skill"
            })
        elif event_type == TYPE_SKILL_HEAL:
            output.append({
                "text": _format_heal_event(m),
                "event_type": "buff"
            })

    _pending_events.clear()
    _window_start_time = current_time
    return output


func clear() -> void:
    _pending_events.clear()
    _window_start_time = 0.0


func _format_damage_event(event: Dictionary) -> String:
    var source: String = str(event.get("source_name", "???"))
    var target: String = str(event.get("target_name", "???"))
    var value: int = int(event.get("value", 0))
    var count: int = int(event.get("count", 1))
    var skill: String = str(event.get("skill_name", ""))
    var team_prefix: String = _team_marker(int(event.get("source_team", 0)))
    
    if not skill.is_empty():
        if count > 1:
            return "%s%s [%s] → %s %d 伤 ×%d" % [team_prefix, source, skill, target, value, count]
        return "%s%s [%s] → %s %d 伤" % [team_prefix, source, skill, target, value]
    
    if count > 1:
        return "%s%s → %s 共 %d 击 %d 伤" % [team_prefix, source, target, count, value]
    return "%s%s → %s %d 伤" % [team_prefix, source, target, value]


func _format_heal_event(event: Dictionary) -> String:
    var source: String = str(event.get("source_name", "???"))
    var target: String = str(event.get("target_name", "???"))
    var value: int = int(event.get("value", 0))
    var skill: String = str(event.get("skill_name", ""))
    var team_prefix: String = _team_marker(int(event.get("source_team", 0)))
    
    if source == target:
        if not skill.is_empty():
            return "%s%s [%s] 恢复 %d" % [team_prefix, source, skill, value]
        return "%s%s 恢复 %d" % [team_prefix, source, value]
    if not skill.is_empty():
        return "%s%s [%s] → %s 治疗 %d" % [team_prefix, source, skill, target, value]
    return "%s%s → %s 治疗 %d" % [team_prefix, source, target, value]


func _format_death_event(event: Dictionary) -> String:
    var dead_name: String = str(event.get("target_name", "???"))
    var killer_name: String = str(event.get("source_name", ""))
    var team_prefix: String = _team_marker(int(event.get("target_team", 0)))
    
    if killer_name.is_empty():
        return "%s%s 阵亡" % [team_prefix, dead_name]
    return "%s%s 被 %s 击杀" % [team_prefix, dead_name, killer_name]


func _format_skill_cast_event(event: Dictionary) -> String:
    var source: String = str(event.get("source_name", "???"))
    var skill: String = str(event.get("skill_name", "技能"))
    var team_prefix: String = _team_marker(int(event.get("source_team", 0)))
    return "%s%s 发动 [%s]" % [team_prefix, source, skill]


func _team_marker(team_id: int) -> String:
    if team_id == 1:
        return "▶"   # 己方
    if team_id == 2:
        return "◀"   # 敌方
    return ""
```

### 4.3 runtime_view flush 改造

```gdscript
# 在 battlefield_hud_runtime_view.gd 中修改

const BATTLE_LOG_FLUSH_INTERVAL_NORMAL: float = 0.25
const BATTLE_LOG_FLUSH_INTERVAL_DENSE: float = 0.50
const BATTLE_LOG_MAX_FLUSH_BATCH: int = 10

var _combat_log_aggregator = null  # 由 presenter 注入


func update_battle_log_view(delta: float) -> void:
    # 面板不可见时完全跳过
    if _refs.battle_log_panel == null or not _refs.battle_log_panel.visible:
        _state.battle_log_flush_accum = 0.0
        return
    
    _state.battle_log_flush_accum += delta
    var interval: float = _resolve_log_flush_interval()
    if _state.battle_log_flush_accum < interval:
        return
    
    # 从聚合器拉取合并后的文案
    if _combat_log_aggregator != null:
        var aggregated: Array[Dictionary] = _combat_log_aggregator.flush_aggregated(
            _state.combat_elapsed
        )
        var batch_count: int = 0
        for entry in aggregated:
            if batch_count >= BATTLE_LOG_MAX_FLUSH_BATCH:
                break
            append_battle_log(str(entry.get("text", "")), str(entry.get("event_type", "info")))
            batch_count += 1
    
    flush_battle_log(true)


func _resolve_log_flush_interval() -> float:
    var total_units: int = 0
    if _state != null:
        total_units = _state.ally_deployed.size() + _state.enemy_deployed.size()
    if total_units >= DENSE_LOG_UNIT_THRESHOLD:
        return BATTLE_LOG_FLUSH_INTERVAL_DENSE
    return BATTLE_LOG_FLUSH_INTERVAL_NORMAL
```

## 7 兜底与边界情况

| 场景 | 处理 |
|------|------|
| 战斗中切到结果期 | `flush_aggregated()` 强制清空，全部写入日志 |
| 重开战场（F7） | `clear()` 环形缓冲 + 清空 RichTextLabel |
| 单位 instance 已失效 | 聚合器只记名字字符串，不持有 Node 引用 |
| 战斗回放 | 聚合器可序列化为 JSON Array 用于回放日志 |
| 日志行数过多（>200 行） | flush 时检查行数 > 上限就全量 clear + 重写最近 N 条 |

## 8 不推荐的方案（及原因）

| 方案 | 为什么不用 |
|------|-----------|
| 每条伤害都实时 append | 50v200 场景下每秒 115+ 次 UI 写入，直接拉崩 FPS |
| 用 Label 替代 RichTextLabel | 无法做颜色区分，信息密度低 |
| 日志全部用飘字（FloatingText） | 250 个单位同时飘字 → GPU overdraw 爆表 |
| 用 ItemList 替代 RichText | 自定义排版困难，且自身在 >100 item 时也有性能问题 |
| 每帧都 clear + rebuild 全部文案 | BBCode 全量解析开销随行数线性增长 |

---

## 9 关键结论

1. **不要逐事件写 UI**：50v200 场景每秒 100+ 事件，必须聚合后再写。
2. **结构化优先**：采集层只写 Dictionary 到缓冲区，不做任何 String 拼接。
3. **节流 + 批量**：flush 间隔提升到 0.25~0.5 秒，每次最多 10 条聚合行。
4. **面板不可见则完全跳过**：最简单的优化，立刻生效。
5. **环形缓冲免搬移**：替代 Array remove_at(0) 的 O(n) 开销。
6. **合并同源同目标**：显著减少重复信息，提高可读性和性能双赢。

func _sample_log_metrics() -> void:
    if _combat_log_aggregator == null:
        return
    # 从聚合器读取累计计数（需在聚合器中暴露）
    _log_events_pushed = int(_combat_log_aggregator.get("_total_pushed_count"))
    _log_lines_output = int(_combat_log_aggregator.get("_total_output_count"))
    _log_flush_count = int(_combat_log_aggregator.get("_total_flush_count"))


# 在报告中增加日志维度：
func _emit_log_report() -> void:
    print("log_metrics events_pushed=%d lines_output=%d flush_count=%d" % [
        _log_events_pushed, _log_lines_output, _log_flush_count
    ])
    print("log_metrics richtext_appends=%d max_batch=%d" % [
        _log_richtext_append_calls, _log_max_flush_batch
    ])
    var compression_ratio: float = 0.0
    if _log_events_pushed > 0:
        compression_ratio = float(_log_lines_output) / float(_log_events_pushed)
    print("log_compression ratio=%.3f (lower is better, target < 0.5)" % compression_ratio)
