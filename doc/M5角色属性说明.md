# M5角色属性机制说明 (Unit Attributes)

本文档总结了 M5 阶段（基于模块化重构后的战斗核心）角色各项基本属性（`base_stats` 和 `runtime_stats`）在战斗引擎中的具体定位及计算公式。AI 在扩充游戏内容（装备/功法/羁绊）时，务必参考此文档设计数值权重。

---

## 一、十大基础属性概览

在 `unit_data.gd` 中定义的默认十大基础属性及其缩写分别如下：

1.  **HP (`hp`)**：最大生命值
2.  **MP (`mp`)**：最大内力量
3.  **ATK (`atk`)**：外功（攻击力）
4.  **IAT (`iat`)**：内功（内力攻击）
5.  **DEF (`def`)**：外防（防御力）
6.  **IDR (`idr`)**：内防（内功抗性）
7.  **SPD (`spd`)**：身法（速度）
8.  **WIS (`wis`)**：悟性（暴击与破防倾向）
9.  **RNG (`rng`)**：射程（攻击范围距离）
10. **MOV (`mov`)**：轻功（移动步幅/寻路权重）

---

## 二、属性在战斗公式中的核心作用

以下机制均通过 `scripts/unit/unit_combat.gd` 中的战斗逻辑及周边体系产生实际效果：

### 1. 生命与生存 (HP / DEF / IDR)

-   **生命值 (`hp`)**：决定角色的 `max_hp`。战斗开始时血量为最大值，归零则死亡。部分特效如斩杀线、生命回复、百分比扣血等均强依赖此数值的上限。
-   **减伤公式**：伤害结算采用护甲公式：`Damage = Offense * (100 / (100 + 防御值))`。
    -   面对**外伤 (external)** 时，采用 `DEF` 计算减伤。
    -   面对**内伤 (internal)** 时，采用 `IDR` 计算减伤。

### 2. 内力与技能 (MP / IAT)

-   **内力池 (`mp`)**：决定角色的 `max_mp`。
    -   **初始内力**：开场默认为最大内力的 35%。
    -   **技能消耗 (`skill_mp_cost`)**：动态计算为 `max_mp * 0.6`，但限制在 `20 ~ 120` 之间。
    -   **回复途径**：每秒自然回复（基础 2 点）、普攻命中回复（15点）、受击回复（10点）。当内力 `current_mp >= skill_mp_cost` 时且未被沉默，下一击必定为技能攻击。
-   **内功/外功自适应**：系统通过对比 `ATK` 和 `IAT` 自动决定当前出手造成的伤害类型：
    -   释放技能时：优先使用内功伤害轨道（如果 `iat >= atk * 0.6`），打出的为**内伤**，受 `IDR` 抵抗。
    -   非技能（普攻）时：优先使用外功伤害轨道（如果 `atk >= iat * 0.8`），打出的为**外伤**，受 `DEF` 抵抗。

### 3. 三维进阶机制 (SPD / WIS)

-   **身法 (`spd`)**：敏捷与速度。
    -   **攻击频率**：攻击间隔公式为 `1.8 - spd / 120.0` 秒（限制在 0.24 ~ 2.2 秒），即 `SPD` 越高，普攻速度越快。
    -   **闪避率 (`dodge_rate`)**：公式为 `(受击方 SPD - 攻击方 SPD) / (受击方 SPD + 100)`，硬性上限 40%（加上装备特效额外提升后最多 75%）。即身法高的人有几率闪避身法低的人的攻击。
-   **悟性 (`wis`)**：暴击与爆发。
    -   **暴击率 (`crit_rate`)**：基础 5% + `wis * 0.1%`。例如 100 悟性提供 10% 额外暴击率（硬上限 80%）。
    -   **暴伤倍率 (`crit_multiplier`)**：基础 1.5倍 + `wis * 0.05%`。例如 100 悟性使暴击伤害从 1.5倍变为 1.55倍。

### 4. 空间与寻路 (RNG / MOV)

-   **射程 (`rng`)**：决定单位是近战还是远程。
    -   底层严格换算为“**六角格子距离**”（最终射程 = `rng` + `range_add` 特效修饰）。
    -   在战斗管理器中，如果目标在 `rng` 格内，判定为可攻击位置；否则寻路逻辑必须推动单位向前靠近。
-   **移动 (`mov`)**：（在寻路与物理移动组件中生效）
    -   决定单位每逻辑帧/每秒跨越格子的速度和最大穿行的格子数意愿。高速移动的角色更易切入后排或拉断敌方仇恨连线。

## 三、战斗中产生的扩展子属性（修饰层）

除了十项主属性外，角色的战斗组件 `UnitCombat` 还常驻了一个 `_external_modifiers` （外部修正层）用于对接装备和功法特效，其部分子属性补充机制如下：

-   `damage_reduce_percent`：跨越外防/内防计算的最终百发比免伤。
    -   `实际受击百发比 = 100% - damage_reduce_percent`。
-   `damage_amp_percent` / `damage_amp_vs_debuffed`：最终伤害放大倍率（比如全伤害加成，对异常状态者追击加成等）。
    -   在防御公式、暴击之后，直接乘算在最终输出期望上。
-   `vampire`：百发比生命偷取（只对外功/非技能攻击生效，直接恢复造成伤害该比例的HP）。
-   `thorns_percent` / `thorns_flat`：接触反伤（同样只对受到外功/非技能攻击时生效，向攻击者反弹真实伤害）。
-   `execute_threshold`：提供特定斩杀线被动（如 0.15 代表生命只有 15% 的目标，对其伤害会直接获得双倍增幅）。
-   `healing_amp`：受疗/施加治疗的效果提升值。

## 设计小贴士 (对于 AI 扩展设计的建议)

1.  **刺客类角色设计**：应堆砌高 `SPD`（保证极短攻击间隔与闪避）和高 `WIS`（高暴击组合），但降低 `DEF` 和 `IDR`，强化 `MOV`。
2.  **法师/内功类设计**：需要极高的 `MP`（因为技能释放门槛通常为内力最大值的 60%）和极高的 `IAT`，普攻阶段较为疲软。内力大等于技能门槛高，应当辅以增加回蓝（内力命中恢复/击杀恢复）的特效。
3.  **坦克/前排类设计**：同时堆高 `HP` 和 `DEF`/`IDR`，通常配合 `thorns`(反伤) 或者 `damage_reduce_percent`(免伤) 特效食用。较低的 `SPD` 使其不仅出手慢，更容易被高 `SPD` 敌人打满命中。
