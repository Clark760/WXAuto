# M5角色属性说明

## 1. 属性集合（当前）

`base_stats` 现在只包含 9 项：

- `hp` 最大生命
- `mp` 最大内力
- `atk` 外功
- `iat` 内功
- `def` 外防
- `idr` 内防
- `spd` 速度
- `rng` 射程（格）
- `wis` 悟性

## 2. 升星规则（已修正）

升星倍率仍为：

- 1星：`1.0x`
- 2星：`1.8x`
- 3星：`3.0x`

但只作用于以下 6 项：

- `hp/mp/atk/iat/def/idr`

不会随升星变化的属性：

- `rng/spd/wis`

因此，`rng/spd/wis` 的基础值在升星后保持不变。

## 3. 属性边界与约束

- `rng`：最小值 `1`，不设上限。
- `spd`：最小值 `0`。
- `wis`：最小值 `0`。
- 其余基础属性最小值 `0`（其中 `hp` 在运行态仍会被保证为正值用于战斗）。

运行时统一收敛（`GongfaManager._clamp_runtime_stats`）：

- 所有属性先夹到 `>= 0`
- `hp` 与 `rng` 额外保证 `>= 1`

## 4. 射程（RNG）生效规则

`UnitCombat.get_attack_range_cells()` 当前规则：

- 先计算：`rng + range_add`
- 再取整：`int(...)`
- 最终只保证 `>= 1`
- 不做上限裁剪

因此，高 `rng` 会直接转化为更远的可攻击格数，不会被战斗组件硬上限截断。
`rng` 的变更来源仅限 effect 体系（例如 `stat_add/stat_percent/range_add`），不由升星与关卡倍率改写。

## 5. SPD / WIS 生效公式

### SPD

- 攻击间隔基础式：
`base_interval = clamp(1.8 - spd / 120.0, 0.24, 2.2)`
- 叠加攻速修正后：
`attack_interval = clamp(base_interval * (1 - attack_speed_bonus), 0.08, 3.0)`
- 闪避基础：
`clamp((target_spd - attacker_spd) / (target_spd + 100), 0, 0.4)`
  再叠加 `dodge_bonus`，最终夹到 `[0, 0.75]`。

### WIS

- 暴击率：
`crit_rate = clamp(0.05 + wis * 0.001 + crit_bonus, 0.05, 0.8)`
- 暴伤倍率：
`crit_multiplier = 1.5 + wis * 0.0005 + crit_damage_bonus`
