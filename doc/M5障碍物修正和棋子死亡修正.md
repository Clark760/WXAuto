# M5 障碍物修正和棋子死亡修正

> **涉及文件**：`obstacle_manager.gd`、`battlefield.gd`、`sprite_animator.gd`

---

## 问题一：障碍物位置没有和格子严格对齐

### 根因

`obstacle_manager._spawn_visual()`（行 116-127）调用 `hex_grid.get_hex_points_local(cell)` 获取六角形顶点。

`hex_grid.get_hex_points_local(cell)` 的实现（hex_grid.gd 行 86-87）：
```gdscript
func get_hex_points_local(cell: Vector2i) -> PackedVector2Array:
    return _build_hex_points(axial_to_local(cell))
```

`_build_hex_points(center)` 返回的是以 `center` 为中心的 6 个绝对点（hex_grid.gd 行 168-174）：
```gdscript
var point: Vector2 = center + Vector2(cos(angle), sin(angle)) * hex_size
```

这些点的坐标是在 **hex_grid 的局部坐标系**下的绝对坐标（包含 `origin_offset`）。

**问题**：`Polygon2D` 被添加到 `_visual_layer`（ObstacleLayer），而 `_visual_layer` 是通过 `configure_visual_parent(parent_node)` 挂到 `parent_node` 下的。如果 `parent_node` 不是 hex_grid 本身，而是 battlefield 或其他节点，`_visual_layer` 的局部坐标系与 hex_grid 的不一致 → 多边形位置偏移。

**即使 `parent_node == hex_grid`**，也可能存在问题：hex_grid 继承自 Node2D（行 1），如果 hex_grid.position 非零（例如被 battlefield 移动过），`_visual_layer` 作为 hex_grid 的子节点会叠加 hex_grid 的 transform，而 `get_hex_points_local` 返回的点已包含 `origin_offset` 但不含 hex_grid.position → 正确。但如果 `_visual_layer` 挂在 hex_grid 的**父节点**上（例如 battlefield），那 `origin_offset` 对但 hex_grid.position 偏移没被加上 → 偏移。

### 修改方案

**最稳妥方案：用世界坐标确定 Polygon2D 的位置**

将 `_spawn_visual` 改为：先获取格心的世界坐标作为 Polygon2D 的 position，再用相对于格心的六角形顶点（偏移量）作为 polygon：

```gdscript
# obstacle_manager.gd — _spawn_visual 重写（行 116-127）：
func _spawn_visual(hex_grid: Node, obstacle_type: String, cell: Vector2i) -> void:
    if _visual_layer == null or not is_instance_valid(_visual_layer):
        return

    var poly := Polygon2D.new()

    # 获取格心世界坐标
    var center_world: Vector2 = hex_grid.call("axial_to_world", cell)
    # 将世界坐标转为 _visual_layer 的局部坐标
    poly.position = _visual_layer.to_local(center_world)

    # 用以格心为原点的局部六角形顶点
    var hex_size: float = float(hex_grid.get("hex_size"))
    var points := PackedVector2Array()
    for i in range(6):
        var angle: float = deg_to_rad(float(i) * 60.0 - 30.0)
        points.append(Vector2(cos(angle), sin(angle)) * hex_size)
    poly.polygon = points

    poly.color = OBSTACLE_COLORS.get(obstacle_type, Color(0.62, 0.42, 0.24, 0.72))
    poly.z_index = -2
    _visual_layer.add_child(poly)
```

**关键改动**：
1. `poly.position = _visual_layer.to_local(center_world)` — 将格心世界坐标转为 visual_layer 的局部坐标
2. 六角形顶点改为**相对于原点**的局部点（不包含格心偏移），这样无论 `_visual_layer` 挂在哪个节点下都能正确对齐

## 问题二：角色死亡后棋子不消失

### 根因

`_play_death()`（sprite_animator.gd 行 399-412）正确执行了死亡动画：
- scale 缩小到 0.2
- modulate.a 渐变到 0.0（透明）
- 完成后 `_stop_tween()` 但**不切回 IDLE**（行 410 注释："死亡状态通常由外部管理"）

**至此单位不可见，行为正确。**

**但**：`battlefield.gd` 行 255-265 的 `reset_all_units_to_idle()` 遍历 `_ally_deployed` 和 `_enemy_deployed` 中的**所有单位**，包括死亡单位：

```gdscript
func reset_all_units_to_idle() -> void:
    var units: Array[Node] = []
    units.append_array(_collect_units_from_map(_ally_deployed))
    units.append_array(_collect_units_from_map(_enemy_deployed))
    for unit in units:
        if not _is_valid_unit(unit):
            continue
        if unit.has_method("reset_visual_transform"):
            unit.call("reset_visual_transform")  # ← 重置 alpha 为 1.0！
        unit.call("play_anim_state", 0, {})      # ← 切 IDLE，normalize 又把 alpha 设 1.0！
```

死亡单位仍在 `_deployed` 映射中（`_handle_unit_death` 只调 `_vacate_unit` 不从部署映射中移除），所以 `reset_all_units_to_idle` 会重新让死亡单位可见。

**另一个触发路径**：`stop_battle()` 中是否对死亡单位调了 `play_anim_state`？检查当前 `stop_battle` 逻辑——如果 stop_battle 遍历 `_all_units` 而死亡单位已被 `_remove_unit_runtime_entry` 移除，则不会被遍历到。但如果 stop_battle 使用 `_enemy_deployed`/`_ally_deployed`（非 `_all_units`），就会碰到死亡单位。

### 修改方案

**修复 1：`reset_all_units_to_idle` 跳过死亡单位**

```gdscript
# battlefield.gd — reset_all_units_to_idle() 修改（行 255-265）：
func reset_all_units_to_idle() -> void:
    var units: Array[Node] = []
    units.append_array(_collect_units_from_map(_ally_deployed))
    units.append_array(_collect_units_from_map(_enemy_deployed))
    for unit in units:
        if not _is_valid_unit(unit):
            continue
        # M5-FIX: 跳过已死亡的单位，避免重新显示死亡棋子
        var combat: Node = unit.get_node_or_null("Components/UnitCombat")
        if combat != null and not bool(combat.get("is_alive")):
            continue
        if unit.has_method("reset_visual_transform"):
            unit.call("reset_visual_transform")
        unit.call("play_anim_state", 0, {})
```

**修复 2：`stop_battle` 中处理死亡单位时跳过动画重置**

检查 `combat_manager.stop_battle()` 中是否有遍历所有单位设置动画的逻辑，如有同样需要跳过死亡单位。当前 `_handle_unit_death` 已调用 `play_anim_state(5)` DEATH 动画，`stop_battle` 不应覆盖该状态。

需 AI 确认 `stop_battle` 的存活单位遍历是否使用 `_all_units`（死亡单位已移除）还是 `_ally_deployed`/`_enemy_deployed`（死亡单位仍在）。如果使用后者，也需要加 `is_alive` 检查。

**修复 3（防御性）：`_play_death` 完成后隐藏单位**

在 DEATH 动画完成回调中直接隐藏单位节点，从根源防止任何后续代码让其重新显示：

```gdscript
# sprite_animator.gd — _play_death() 行 409-412 修改：
_tween.finished.connect(func() -> void:
    _stop_tween()
    # M5-FIX: 死亡动画完成后隐藏整个单位，防止后续状态切换意外显示
    var unit_root: Node = _target.get_parent() if _target != null else null
    if unit_root != null and unit_root is CanvasItem:
        (unit_root as CanvasItem).visible = false
)
```

同时在 `enter_combat()` 中确保重新显示：

```gdscript
# unit_base.gd — enter_combat() 中新增：
func enter_combat() -> void:
    visible = true  # M5-FIX: 恢复上局死亡后被隐藏的可见性
    is_in_combat = true
    # ... 其余不变 ...
```
