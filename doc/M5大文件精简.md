# M5 大文件精简与拆分方案

> **目标读者**：AI 代码生成器。本文档定义四个 1000+ 行文件的拆分策略，遵循**高内聚低耦合**原则。

---

## 一、文件现状

| 文件 | 行数 | 主要职责 | 继承关系 |
|------|------|----------|----------|
| `battlefield_runtime.gd` | ~2114 | 棋盘渲染、拖拽、单位部署、MultiMesh、摄像机 | Node2D 基类 |
| `battlefield.gd` | ~2476 | 商店、经济、关卡、回收、仓库、战斗启动 | 继承 battlefield_ui.gd |
| `combat_manager.gd` | ~1472 | 逻辑帧驱动、占格、流场、寻敌、攻击、死亡 | Node |
| `gongfa_manager.gd` | ~1169 | 功法被动、技能触发、Buff事件、联动、装备 | Node |

---

## 二、拆分总原则

1. **按职责域拆**：每个新文件只干一件事
2. **用组合非继承**：拆出的模块以子节点或 RefCounted 对象形式挂载，由原文件持有引用
3. **最少改接口**：外部调用方（如 `battlefield.gd` 调 `combat_manager.start_battle()`）不变，拆分只在内部进行
4. **GDScript 限制**：不支持 partial class，只能用组合模式拆分

## 八、实施顺序

拆分优先级基于"改动风险低 + 收益高"排序：

| 步骤 | 目标 | 风险 |
|------|------|------|
| 1 | `combat_metrics.gd` — 纯数据搬迁，零逻辑耦合 | 极低 |
| 2 | `cell_occupancy.gd` — 接口清晰，独立性强 | 低 |
| 3 | `combat_targeting.gd` — 依赖 CellOccupancy | 低 |
| 4 | `combat_pathfinding.gd` — 依赖 CellOccupancy + FlowField | 低 |
| 5 | `trigger_engine.gd` — 从 gongfa_manager 拆出触发逻辑 | 中 |
| 6 | `passive_applier.gd` — 从 gongfa_manager 拆出被动应用 | 中 |
| 7 | `battlefield_renderer.gd` — 纯渲染，不影响逻辑 | 低 |
| 8 | `unit_deploy_manager.gd` — 核心部署逻辑 | 中 |
| 9 | `drag_controller.gd` — 依赖 deploy_manager | 中 |
| 10 | `shop_controller.gd` — UI 逻辑独立 | 低 |
| 11 | `stage_bridge.gd` — 关卡逻辑独立 | 中 |
| 12 | `inventory_controller.gd` — UI 逻辑独立 | 低 |
| 13 | `recycle_controller.gd` — 最小模块 | 极低 |

## 九、拆分模板（示例：`cell_occupancy.gd`）

### 步骤 1：创建新文件

```gdscript
# scripts/combat/cell_occupancy.gd
extends RefCounted
class_name CellOccupancy

var _hex_grid: Node = null
var _cell_occupancy: Dictionary = {}
var _unit_cell: Dictionary = {}
var _static_blocked_cells: Dictionary = {}

func setup(hex_grid: Node, static_blocked: Dictionary) -> void:
    _hex_grid = hex_grid
    _static_blocked_cells = static_blocked

func clear() -> void:
    _cell_occupancy.clear()
    _unit_cell.clear()

# 从 combat_manager.gd 原样搬迁以下函数（去掉 _ 前缀改为 public）：
# occupy_cell, vacate_cell, vacate_unit, is_cell_free,
# get_unit_cell, find_nearest_free_cell, cell_key_int, parse_cell_key_int,
# validate_occupancy, neighbors_of
```

### 步骤 2：修改 `combat_manager.gd`

```gdscript
# 新增成员变量：
var _occupancy: CellOccupancy = CellOccupancy.new()

# _ready 或 start_battle 中初始化：
_occupancy.setup(_hex_grid, _static_blocked_cells)

# 所有内部调用替换：
# _occupy_cell(cell, unit)    → _occupancy.occupy_cell(cell, unit)
# _is_cell_free(cell)         → _occupancy.is_cell_free(cell)
# _get_unit_cell(unit)        → _occupancy.get_unit_cell(unit)
# ...
```

### 步骤 3：验证

运行一场完整战斗，确认占格、移动、死亡、结算行为不变。

## 十、不拆分的函数

以下函数因为耦合多个子系统，应保留在原文件中：

| 文件 | 保留函数 | 原因 |
|------|----------|------|
| `combat_manager.gd` | `_run_unit_logic` | 编排攻击/移动/寻敌，是核心调度 |
| `combat_manager.gd` | `_logic_tick` | 编排预扫描/流场/逻辑帧 |
| `battlefield_runtime.gd` | `_ready`, `_set_stage` | 生命周期入口 |
| `battlefield.gd` | `_ready`, `_set_stage`, `_start_combat` | 编排多模块协作 |
| `gongfa_manager.gd` | `_process`, `prepare_battle` | 编排触发/Buff/联动 |
