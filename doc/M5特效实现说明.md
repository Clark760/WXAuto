# M5特效实现说明（以当前代码为准）

本文档对应以下实现：
- `scripts/gongfa/effect_engine.gd`
- `scripts/gongfa/tag_linkage_resolver.gd`

仅列出当前代码中真实可执行的 `op`。旧文档中的 `burn_ground`、`hp_regen` 等已不在执行链路中。

## 1. 执行入口与上下文

- 被动效果入口：`apply_passive_effects(runtime_stats, modifier_bundle, effects, stack_multiplier)`
- 主动效果入口：`execute_active_effects(source, target, effects, context)`

常用 `context` 键：
- `all_units`：范围检索、链式伤害、最低血友军等
- `buff_manager`：Buff/Debuff、净化、驱散、偷取、战场危险区
- `combat_manager`：格子坐标、拉拽/击退/换位/瞬移、临时地形
- `hex_grid` / `hex_size`：格距换算、半径搜索
- `battlefield`：召唤单位
- `gongfa_manager`：`tag_linkage_branch` 分支求值
- `vfx_factory`：播放特效
- `battle_elapsed`：逻辑时间（标记、控制持续）

## 4. `tag_linkage_branch` 详细配置

### 4.1 顶层字段

| 字段 | 说明 |
|---|---|
| `range` | 搜索半径（格）。`0` 只看自身与自身格地形。 |
| `include_self` | 是否把自身作为候选提供者。 |
| `source_types` | 全局来源类型：`trait` / `gongfa` / `equipment` / `terrain` / `unit`。 |
| `team_scope` | 全局阵营过滤：`ally` / `enemy` / `all`（默认 `ally`）。 |
| `count_mode` | 计数方式：`provider`(默认)、`unit`、`occurrence`。 |
| `queries` | 查询列表。 |
| `cases` | 分支列表（按顺序匹配）。 |
| `else_effects` | 无 case 命中时执行的 effect 列表。 |
| `stop_after_first_case` | 是否命中首个 case 后停止（默认 `true`）。 |

### 4.2 query 字段

| 字段 | 说明 |
|---|---|
| `id` | 查询 id，供 case 条件引用。 |
| `tags` | 要匹配的 tag 列表。 |
| `tag_match` | `any` 或 `all`（默认 `any`）。 |
| `source_types` | 覆盖全局来源类型（可选）。 |
| `team_scope` | 覆盖全局阵营范围（可选）。 |
| `origin_scope` | `all` / `self` / `nearby`。 |

### 4.3 case 字段

| 字段 | 说明 |
|---|---|
| `id` | 分支标识（调试/日志用）。 |
| `all` | 全满足条件列表。 |
| `any` | 任一满足条件列表（可选）。 |
| `effects` | 命中后执行的 effect 列表。 |

条件项字段：
- `query_id`：引用 query
- `min_count` 或 `min`
- `max_count` 或 `max`

## 5. 实务注意

- 同名 `op` 在被动/主动中可能语义不同：`mp_regen_add` 被动是“每秒回蓝修正”，主动是“即时回蓝”。
- 所有范围判定依赖 `all_units`；未提供会导致范围类 op 命中为空。
- 位移/地形类强依赖 `combat_manager`、`hex_grid`；缺失时会失败或退化到世界坐标位移。
- `immunity_self` 必须提供 `buff_id`；缺失时不会执行免疫效果，仅输出警告。
