# 战场 UI 架构设计（面向 AI 代码生成）

> 本文档定义战斗场景的完整 UI 层级、坐标系统、布局规则和阶段行为。
> 所有战场相关代码**必须严格遵循**本文档的层级结构与坐标规则。

---

## 一、设计原则

| 原则 | 说明 |
|------|------|
| **世界/UI 完全分离** | 战场世界（棋盘+棋子+特效）与 UI 面板在**不同坐标空间**，互不干扰 |
| **单点缩放** | 整个战场世界只通过**一个容器节点**的 transform 控制缩放/平移 |
| **本地坐标优先** | 战场世界内部所有定位使用 `position`，**禁止 `global_position` 赋值** |
| **阶段驱动显隐** | UI 元素根据游戏阶段（布阵/交锋/结算）自动切换可见性和交互性 |
| **响应式布局** | 所有 UI 区域基于窗口尺寸动态计算，支持任意分辨率 |

## 三、坐标系统

### 3.1 三套独立坐标空间

```
┌─────────────────────────────────────────────────────┐
│  坐标空间 1：屏幕坐标（Screen Space）                 │
│  - CanvasLayer 内所有 UI 使用                         │
│  - 鼠标 event.position                               │
│  - 不受 WorldContainer 影响                           │
├─────────────────────────────────────────────────────┤
│  坐标空间 2：世界坐标（World Space）                   │
│  - WorldContainer 内部本地坐标                        │
│  - HexGrid / UnitLayer / VfxLayer 共用                │
│  - 缩放/平移通过 WorldContainer.transform 统一管理     │
├─────────────────────────────────────────────────────┤
│  坐标空间 3：格子坐标（Grid Space）                    │
│  - 六角格轴向坐标 Vector2i(q, r)                      │
│  - 通过 HexGrid 的转换函数与世界坐标互转               │
└─────────────────────────────────────────────────────┘
```

### 3.2 坐标转换规则

| 转换方向 | 方法 |
|---------|------|
| 屏幕 → 世界 | `world_container.to_local(screen_pos)` 或 `world_container.get_global_transform().affine_inverse() * screen_pos` |
| 世界 → 屏幕 | `world_container.to_global(world_pos)` |
| 世界 → 格子 | `hex_grid.world_to_axial(world_pos)` |
| 格子 → 世界 | `hex_grid.axial_to_world(cell)` |
| 屏幕 → 格子 | 先屏幕→世界，再世界→格子（两步） |

### 3.3 各节点坐标使用规范

| 所在节点 | 定位方式 | 说明 |
|---------|---------|------|
| UnitBase（战场棋子） | `unit.position = hex_grid.axial_to_world(cell)` | 本地坐标，自动随 WorldContainer 缩放 |
| VFX 粒子/伤害数字 | `node.position = attacker.position` | 本地坐标，与棋子同一空间 |
| HUD 面板/标签 | 用 Container 布局，锚点定位 | 屏幕坐标，不受缩放影响 |
| BenchSlot 棋子 | Container 子项自动排列 | 屏幕坐标，不在 WorldContainer 内 |

### 3.4 禁止事项

```gdscript
# ❌ 禁止在 WorldContainer 子节点上使用 global_position 赋值
unit.global_position = some_pos
particles.global_position = attacker.global_position

# ✅ 正确用法
unit.position = hex_grid.axial_to_world(cell)
particles.position = attacker.position
```

## 四、WorldContainer 视角控制

### 4.1 核心变量

```gdscript
@onready var world_container: Node2D = $WorldContainer

var _world_zoom: float = 1.0       # 当前缩放倍率
var _world_offset: Vector2 = Vector2.ZERO  # 当前平移偏移
var _is_panning: bool = false       # 右键拖拽中

const ZOOM_MIN: float = 0.4
const ZOOM_MAX: float = 2.5
const ZOOM_STEP: float = 0.1
const PAN_SPEED: float = 540.0     # WASD 平移速度（像素/秒）
```

### 4.2 核心函数

```gdscript
# 唯一的 transform 应用入口——只操作 WorldContainer
func _apply_world_transform() -> void:
    world_container.position = _world_offset
    world_container.scale = Vector2.ONE * _world_zoom

# 鼠标滚轮缩放（锚点跟随鼠标）
func _zoom_at(screen_pos: Vector2, factor: float) -> void:
    var old_zoom := _world_zoom
    _world_zoom = clampf(old_zoom * factor, ZOOM_MIN, ZOOM_MAX)
    var world_point := (screen_pos - _world_offset) / old_zoom
    _world_offset = screen_pos - world_point * _world_zoom
    _apply_world_transform()

# 右键拖拽平移
func _pan(relative: Vector2) -> void:
    _world_offset += relative
    _apply_world_transform()

# 空格键复位
func _reset_view() -> void:
    _world_zoom = 1.0
    _world_offset = Vector2.ZERO
    _apply_world_transform()
```

### 4.3 输入处理

| 操作 | 触发条件 | 效果 | 适用阶段 |
|------|---------|------|---------|
| 鼠标滚轮 | 任何阶段 | 缩放，锚定鼠标位置 | 全阶段 |
| 右键拖拽 | 任何阶段 | 平移视角 | 全阶段 |
| WASD 键 | 任何阶段 | 持续平移 | 全阶段 |
| 空格键 | 非布阵 | 视角复位 / 切换全场↔半场 | 交锋+结算 |
| 双击棋子 | 交锋中 | 镜头聚焦跟随该单位 | 交锋 |

## 五、HexGrid 棋盘层设计

### 5.1 棋盘参数

| 参数 | 值 | 说明 |
|------|-----|------|
| `grid_width` | 32 | 横向格子数 |
| `grid_height` | 16 | 纵向格子数 |
| `hex_size` | 基于窗口动态计算 | 六角格半径（px），fit 到可用区域 |
| `origin_offset` | 动态居中 | 网格在 WorldContainer 内的位置 |
| 己方部署区 | 列 0-15（前16列） | 约 128 格 |
| 敌方区域 | 列 16-31（后16列） | 约 256 格 |

### 5.2 棋盘自适应布局

```gdscript
# 棋盘应 fit 到窗口中央，留出顶部和底部 UI 区域
func _refit_hex_grid() -> void:
    var viewport_size := get_viewport().get_visible_rect().size
    var top_reserved := 60.0     # 顶部状态栏
    var bottom_reserved := 220.0 # 底部面板（布阵期）
    var margin := 20.0
    
    var available_w := viewport_size.x - margin * 2
    var available_h := viewport_size.y - top_reserved - bottom_reserved
    
    # 根据可用空间反算 hex_size，使棋盘完整 fit
    var fit_hex_size := _calculate_fit_hex_size(available_w, available_h)
    hex_grid.hex_size = fit_hex_size
    
    # 居中偏移
    var board_size := _calculate_board_pixel_size(fit_hex_size)
    hex_grid.origin_offset = Vector2(
        (viewport_size.x - board_size.x) * 0.5,
        top_reserved + (available_h - board_size.y) * 0.5
    )
```

### 5.3 棋盘视觉层次

| 层 | z_index | 内容 |
|----|---------|------|
| 网格线 | 0 | 六角格边框 |
| 部署区高亮 | 1 | 己方可部署区域半透明填充 |
| 悬停高亮 | 2 | 鼠标当前悬停格子高亮 |
| 拖拽预览 | 3 | 拖拽中目标格子绿/红色提示 |

## 六、UnitLayer 棋子层设计

### 6.1 棋子节点结构

```
UnitBase (Node2D)                    ← position = 格子世界坐标
├── SpriteContainer (Node2D)         ← SpriteAnimator 操作此节点
│   └── Sprite2D                     ← 战斗精灵图
├── HPBar (TextureProgressBar)       ← 血条，固定在精灵图上方
├── StarIcons (HBoxContainer)        ← 星级图标，固定在血条上方
└── TeamIndicator (ColorRect)        ← 阵营色条（红/蓝），精灵图下方
```

### 6.2 棋子缩放规则

| 场景 | 缩放方式 | 说明 |
|------|---------|------|
| 战场部署 | `unit.scale = Vector2.ONE * _unit_scale` | `_unit_scale` 基于 `hex_size` 计算，使棋子适配格子大小 |
| 备战席 | 由 BenchSlot 容器控制尺寸 | 不在 WorldContainer 内，独立尺寸 |
| WorldContainer 缩放 | 自动继承 | 棋子作为子节点自动跟随，无需额外处理 |

### 6.3 棋子 z_index 排序

```gdscript
# 棋子按 y 坐标排序，下方棋子覆盖上方棋子（模拟前后关系）
# 在 UnitBase._process() 中：
func _update_z_order() -> void:
    z_index = int(position.y)
```

## 七、VfxLayer 特效层设计

### 7.1 特效定位规则

```gdscript
# 所有特效都在 VfxLayer 下，与 UnitLayer 同一坐标空间

# 攻击特效：从攻击者 position 到目标 position
func play_attack_vfx(vfx_id: String, from_pos: Vector2, to_pos: Vector2) -> void:
    var particles := _acquire_particles(vfx_id)
    particles.position = from_pos       # ✅ 本地坐标
    particles.rotation = (to_pos - from_pos).angle()

# 伤害数字：在目标 position 上方
func spawn_damage_text(at_pos: Vector2, amount: float) -> void:
    var label := _acquire_label()
    label.position = at_pos + Vector2(0, -20)  # ✅ 本地坐标
    # Tween 操作 position.y（本地属性）
    var tw := create_tween()
    tw.tween_property(label, "position:y", label.position.y - 30, 0.4)
```

### 7.2 特效 z_index

| 特效类型 | z_index | 原因 |
|---------|---------|------|
| 地面效果（阵法光圈） | -10 | 在棋子下方 |
| 攻击粒子 | 100 | 在棋子上方 |
| 伤害数字 | 200 | 最上层 |

**交互规则**：

| 规则 | 说明 |
|------|------|
| **触发** | 鼠标在棋子上停留 **0.3s** 后显示（避免快速扫过时闪烁） |
| **消失** | 鼠标离开棋子 **0.15s** 后隐藏（给出缓冲、避免抖动） |
| **定位** | 跟随鼠标屏幕坐标，偏移 `(16, 16)` |
| **边界翻转** | 距右边缘 < 面板宽度时翻到鼠标左侧；距下边缘同理翻到上方 |
| **战斗中可用** | 交锋期/结算期鼠标悬停同样显示，实时刷新 HP/Buff 等 |
| **缩放适配** | Tooltip 在 CanvasLayer 中，不受世界缩放影响 |

### 8.7 悬浮检测逻辑

```gdscript
# 在 BattleScene._process() 或 _unhandled_input() 中实现

var _hover_unit: Node = null
var _hover_timer: float = 0.0
const HOVER_DELAY: float = 0.3
const HOVER_FADE_DELAY: float = 0.15
var _hover_fade_timer: float = 0.0

func _process(delta: float) -> void:
    var screen_mouse := get_viewport().get_mouse_position()
    var world_mouse := world_container.to_local(screen_mouse)
    var hit_unit := _find_unit_at_world_pos(world_mouse)
    
    if hit_unit != null and hit_unit != _hover_unit:
        # 切换到新棋子
        _hover_unit = hit_unit
        _hover_timer = 0.0
        _hover_fade_timer = 0.0
        unit_tooltip.visible = false
    elif hit_unit != null:
        # 持续悬停
        _hover_timer += delta
        _hover_fade_timer = 0.0
        if _hover_timer >= HOVER_DELAY:
            _show_tooltip(hit_unit, screen_mouse)
    elif _hover_unit != null:
        # 离开棋子
        _hover_fade_timer += delta
        if _hover_fade_timer >= HOVER_FADE_DELAY:
            unit_tooltip.visible = false
            _hover_unit = null

func _show_tooltip(unit: Node, screen_pos: Vector2) -> void:
    _update_tooltip_data(unit)
    # 定位到鼠标旁，含边界翻转
    var offset := Vector2(16, 16)
    var tooltip_size := unit_tooltip.size
    var viewport_size := get_viewport().get_visible_rect().size
    if screen_pos.x + offset.x + tooltip_size.x > viewport_size.x:
        offset.x = -tooltip_size.x - 16
    if screen_pos.y + offset.y + tooltip_size.y > viewport_size.y:
        offset.y = -tooltip_size.y - 16
    unit_tooltip.position = screen_pos + offset
    unit_tooltip.visible = true

func _find_unit_at_world_pos(world_pos: Vector2) -> Node:
    # 遍历 UnitLayer 子节点，检查 contains_point
    for unit in unit_layer.get_children():
        if unit.has_method("contains_point") and unit.contains_point(world_pos):
            return unit
    return null
```

## 十、拖拽交互流程

### 10.1 从备战席到战场

```
1. 鼠标在备战席格子上按下左键
2. 从 BenchSlot 取出棋子数据，创建半透明拖拽预览（屏幕坐标跟随鼠标）
3. 鼠标移动时：
   a. 预览跟随鼠标
   b. 将鼠标屏幕坐标转为世界坐标：world_pos = world_container.to_local(screen_pos)
   c. 将世界坐标转为格子坐标：cell = hex_grid.world_to_axial(world_pos)
   d. 如果 cell 在己方部署区且未占用 → 高亮格子为绿色
   e. 否则 → 高亮格子为红色
4. 松手时：
   a. 如果目标格子合法 → 实例化棋子到 UnitLayer，position = hex_grid.axial_to_world(cell)
   b. 否则 → 棋子返回备战席
5. 销毁拖拽预览
```

### 10.2 从战场到备战席

```
1. 鼠标在战场棋子上按下左键
2. 从 UnitLayer 取出棋子，创建半透明拖拽预览
3. 松手时：
   a. 如果在备战席区域 → 放入备战席
   b. 如果在战场其他合法格子 → 移动到新格子
   c. 否则 → 放回原格子
```

### 10.3 坐标转换关键点

```gdscript
# 拖拽过程中判断鼠标目标
func _get_drop_target(screen_mouse: Vector2) -> Dictionary:
    # 1. 检查是否在底部面板区域（屏幕坐标）
    if bottom_panel.get_global_rect().has_point(screen_mouse):
        return {"type": "bench"}
    
    # 2. 转为世界坐标检查战场格子
    var world_pos := world_container.to_local(screen_mouse)
    var cell := hex_grid.world_to_axial(world_pos)
    if hex_grid.is_inside_grid(cell) and _is_ally_deploy_zone(cell):
        return {"type": "battlefield", "cell": cell, "world_pos": world_pos}
    
    return {"type": "invalid"}
```
